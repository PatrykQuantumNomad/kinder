Purpose: Deliver the batteries-included local registry experience -- users push to `localhost:5001` from the host and pods pull from `localhost:5001` without any manual setup.
Output: Working `installlocalregistry` action package + `create.go` wiring (ContainerdConfigPatches + runAddon).
</objective>

    if !enabled {
        logger.V(0).Infof(" * Skipping %s (disabled in config)\n", name)
        addonResults = append(addonResults, addonResult{name: name, enabled: false})
        return
    }

        addonResults = append(addonResults, addonResult{name: name, enabled: true, err: err})

}

**manifests/local-registry-hosting.yaml** -- Embedded ConfigMap for KEP-1755 dev tool discovery:
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: local-registry-hosting
  namespace: kube-public
data:
  localRegistryHosting.v1: |
    host: "localhost:5001"
    help: "https://kind.sigs.k8s.io/docs/user/local-registry/"
```

**localregistry.go** -- Action implementation with these exact steps:

1. **Package declaration:** `package installlocalregistry` with Apache 2.0 copyright header (same as all other action packages).

2. **Imports:** `embed`, `fmt`, `strings` from stdlib; `actions`, `nodeutils`, `errors`, `exec` from `sigs.k8s.io/kind/pkg/...`. Follow the import grouping convention (stdlib, then third-party, then local).

3. **Embedded manifest:** `//go:embed manifests/local-registry-hosting.yaml` into `var localRegistryHostingManifest string`.

4. **Constants:**
   - `registryName = "kind-registry"` (container name)
   - `registryPort = "5001"` (host-published port)
   - `registryIntPort = "5000"` (container-internal port)
   - `registryImage = "registry:2"` (NOT registry:3 -- v3 deprecated storage drivers, kind ecosystem not migrated)
   - `registryDir = "/etc/containerd/certs.d/localhost:5001"` (certs.d path inside nodes)

5. **hostsTOML constant:** Single-line TOML content pointing at `http://kind-registry:5000`. Use the container name (NOT localhost) because inside node containers localhost is the container's own loopback.
   ```go
   const hostsTOML = `[host."http://kind-registry:5000"]
   `
   ```

6. **NewAction()** returning `&action{}` implementing `actions.Action`.

7. **Execute method** with these sequential steps:
   - `ctx.Status.Start("Installing Local Registry")` + `defer ctx.Status.End(false)`
   - **Detect provider binary:** `binaryName := "docker"` then `ctx.Provider.(fmt.Stringer)` type assertion (same as MetalLB)
   - **Podman rootless warning:** Call `ctx.Provider.Info()`, check `info.Rootless && binaryName == "podman"`, and emit `ctx.Logger.Warn(...)` about manual insecure registry config needed in `/etc/containers/registries.conf`. Do NOT attempt to automate Podman host config.
   - **Step 1 (REG-01) -- Create registry container (idempotent):** Check existence with `exec.Command(binaryName, "inspect", "--format={{.ID}}", registryName).Run()`. If it returns error (container doesn't exist), run `exec.Command(binaryName, "run", "-d", "--restart=always", "--name", registryName, "-p", "127.0.0.1:"+registryPort+":"+registryIntPort, registryImage).Run()`. Wrap errors with `errors.Wrap(err, "failed to create registry container")`.
   - **Step 2 (REG-01) -- Connect to kind network (idempotent):** Inspect current networks via `exec.Command(binaryName, "inspect", "--format", "{{range $k, $v := .NetworkSettings.Networks}}{{$k}} {{end}}", registryName)` using `exec.Output`. Check if "kind" appears in the output. If not, run `exec.Command(binaryName, "network", "connect", "kind", registryName).Run()`. Do NOT just ignore the connect error -- check first to be robust across docker/podman/nerdctl versions.
   - **Step 3 (REG-02) -- Patch containerd certs.d on ALL nodes:** Get `allNodes` via `ctx.Nodes()`. Loop over ALL nodes (not just control-plane). For each node: `node.Command("mkdir", "-p", registryDir).Run()` then `node.Command("tee", registryDir+"/hosts.toml").SetStdin(strings.NewReader(hostsTOML)).Run()`. Use `tee` (not `sh -c "cat >"`) to avoid shell invocation inside node containers. Wrap errors with `errors.Wrapf(err, "... on node %s", node)`.
   - **Step 4 (REG-03) -- Apply ConfigMap:** Get control plane nodes via `nodeutils.ControlPlaneNodes(allNodes)`. Error if none found. Apply the embedded manifest via `controlPlanes[0].Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "apply", "-f", "-").SetStdin(strings.NewReader(localRegistryHostingManifest)).Run()`.
   - **Step 5 -- User notification:** `ctx.Status.End(true)` then log usage instructions:
     ```
     Local Registry:
       Push:  docker push localhost:5001/myimage:latest
       Pull in pods: image: localhost:5001/myimage:latest
     ```
   - Return nil.

**CRITICAL anti-patterns to avoid:**
- Do NOT use `localhost` in hosts.toml -- inside nodes, localhost is the container's loopback
- Do NOT only patch control-plane nodes -- ALL nodes need hosts.toml
- Do NOT restart containerd after writing hosts.toml -- containerd v2 hot-reloads certs.d when config_path is set
- Do NOT use registry:3 -- kind ecosystem has not migrated

Modify `create.go` to add two things: ContainerdConfigPatches injection (pre-provisioning) and runAddon wiring (post-provisioning).

**1. Add import:**

Add `"sigs.k8s.io/kind/pkg/cluster/internal/create/actions/installlocalregistry"` to the import block, after `installmetallb` (alphabetical order within the local imports group).

**2. Inject ContainerdConfigPatches BEFORE p.Provision() (CRITICAL):**

In the `Cluster()` function, AFTER `opts.Config.Validate()` returns and BEFORE `p.Provision(status, opts.Config)` is called (approximately after the "we're going to start creating now" log line), add:

```go
// Inject containerd config_path for local registry (must be before Provision)
if opts.Config.Addons.LocalRegistry {
    opts.Config.ContainerdConfigPatches = append(
        opts.Config.ContainerdConfigPatches,
        `[plugins."io.containerd.grpc.v1.cri".registry]
  config_path = "/etc/containerd/certs.d"
`,
    )
}
```

This MUST happen before `p.Provision()` because containerd config patches are consumed during node provisioning. The addon action runs post-provisioning and cannot modify containerd's config.toml. Without this, hosts.toml files are silently ignored.

**3. Add runAddon call BEFORE MetalLB:**

In the addon section (after `waitforready` and kubeconfig export), add the Local Registry runAddon call BEFORE MetalLB:

```go
runAddon("Local Registry", opts.Config.Addons.LocalRegistry, installlocalregistry.NewAction())
runAddon("MetalLB", opts.Config.Addons.MetalLB, installmetallb.NewAction())
// ... rest of addons unchanged
```

The registry must run before MetalLB because MetalLB may deploy pods that pull images from the registry. The ordering ensures the registry is available for any subsequent addon that might need to pull images.
