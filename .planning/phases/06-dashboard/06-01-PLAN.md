Replace the stub `dashboard.go` with a complete implementation following the Metrics Server action pattern (metricsserver.go) extended with the CoreDNS SetStdout pattern for token reading. The Execute function must:

1. `ctx.Status.Start("Installing Dashboard")` + `defer ctx.Status.End(false)`
2. Get control plane node via `ctx.Nodes()` + `nodeutils.ControlPlaneNodes()` (same as metricsserver.go)
3. Apply the embedded manifest: `node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "apply", "-f", "-").SetStdin(strings.NewReader(headlampManifest)).Run()` — standard apply, NOT server-side (manifest is < 10 KB)
4. Wait for Deployment: `node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "wait", "--namespace=kube-system", "--for=condition=Available", "deployment/headlamp", "--timeout=120s").Run()`
5. Read token from Secret: `node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "get", "secret", "kinder-dashboard-token", "--namespace=kube-system", "-o", "jsonpath={.data.token}").SetStdout(&tokenBuf).Run()`
6. Decode base64 in Go: `base64.StdEncoding.DecodeString(strings.TrimSpace(tokenBuf.String()))`
7. End spinner BEFORE printing: `ctx.Status.End(true)` — the deferred `End(false)` becomes a no-op (confirmed idempotent in cli/status.go)
8. Print dashboard access info via `ctx.Logger.V(0).Info/Infof`:
   ```
   (blank line)
   Dashboard:
     Token: <decoded JWT>
     Port-forward: kubectl port-forward -n kube-system service/headlamp 8080:80
     Then open: http://localhost:8080
   (blank line)
   ```
9. Return nil
