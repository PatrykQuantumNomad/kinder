   - `func (a *action) Execute(ctx *actions.ActionContext) error` implementing:

     Step 1: `ctx.Status.Start("Installing Cert Manager 📜")` and `defer ctx.Status.End(false)`

     Step 2: Get control-plane node -- identical to EnvoyGW:
     ```go
     allNodes, err := ctx.Nodes()
     // ... error check
     controlPlanes, err := nodeutils.ControlPlaneNodes(allNodes)
     // ... error check + empty check
     ```

     Step 3 (CERT-01): Apply cert-manager manifest with --server-side:
     ```go
     node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf",
         "apply", "--server-side", "-f", "-",
     ).SetStdin(strings.NewReader(certManagerManifest)).Run()
     ```
     Error message: "failed to apply cert-manager manifest"
     CRITICAL: Must use `--server-side` because cert-manager CRDs exceed the 256KB `last-applied-configuration` annotation limit. Do NOT use standard `kubectl apply`.

     Step 4 (CERT-02): Wait for all 3 deployments in cert-manager namespace with 300s timeout:
     ```go
     for _, deployment := range []string{
         "cert-manager",
         "cert-manager-cainjector",
         "cert-manager-webhook",
     } {
         node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf",
             "wait", "--namespace=cert-manager",
             "--for=condition=Available",
             "deployment/"+deployment,
             "--timeout=300s",
         ).Run()
         // Error: errors.Wrapf(err, "cert-manager deployment %s did not become available", deployment)
     }
     ```
     CRITICAL: All three must be Available BEFORE applying ClusterIssuer. The webhook validates cert-manager CRs -- applying ClusterIssuer before webhook is ready causes "no endpoints available" errors.

     Step 5 (CERT-03): Apply self-signed ClusterIssuer with standard (NOT --server-side) apply:
     ```go
     node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf",
         "apply", "-f", "-",
     ).SetStdin(strings.NewReader(selfSignedClusterIssuerYAML)).Run()
     ```
     Error message: "failed to apply selfsigned ClusterIssuer"
     Standard client-side apply is correct here -- ClusterIssuer is a small resource.

     Step 6: `ctx.Status.End(true)` and `return nil`
