<objective>
Implement the Metrics Server addon action that installs Metrics Server v0.8.1 with --kubelet-insecure-tls into kube-system during cluster creation, enabling kubectl top and HPA metrics.

Purpose: Metrics Server provides the metrics.k8s.io API that kubectl top and HPA depend on. Kind kubelets use self-signed TLS certificates, so --kubelet-insecure-tls is mandatory. Without this addon, kubectl top returns "Metrics API not available" and HPA cannot scale on CPU/memory.

Output: Working Metrics Server action that replaces the stub -- manifest embedded at build time, applied via kubectl, deployment waited for readiness.
</objective>
