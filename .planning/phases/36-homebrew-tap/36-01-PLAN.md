<task type="auto">
  <name>Task 1: Create homebrew-kinder tap repository and configure GoReleaser cask publishing</name>
  <files>.goreleaser.yaml, .github/workflows/release.yml</files>
  <action>
Three steps in sequence:

1. Create the tap repository using gh CLI:
```bash
gh repo create PatrykQuantumNomad/homebrew-kinder \
  --public \
  --description "Homebrew tap for kinder — kind, but with everything you actually need." \
  --add-readme
```
Verify creation with `gh repo view PatrykQuantumNomad/homebrew-kinder`.

2. Append the `homebrew_casks` section to `.goreleaser.yaml` AFTER the existing `release:` section (at the end of file). Use exactly this config:
```yaml
homebrew_casks:
  - name: kinder
    directory: Casks
    homepage: "https://kinder.patrykgolabek.dev"
    description: "kind, but with everything you actually need."
    skip_upload: auto
    repository:
      owner: PatrykQuantumNomad
      name: homebrew-kinder
      branch: main
      token: "{{ .Env.HOMEBREW_TAP_TOKEN }}"
    commit_author:
      name: goreleaserbot
      email: bot@goreleaser.com
    commit_msg_template: "Brew cask update for {{ .ProjectName }} version {{ .Tag }}"
```

IMPORTANT: Do NOT use `brews:` (deprecated since v2.10). Use `homebrew_casks:` exclusively.
IMPORTANT: `skip_upload: auto` prevents publishing casks for pre-release tags (e.g., v2.0.0-rc1).
IMPORTANT: `directory: Casks` not `Formula` — casks go in Casks/, formulas in Formula/.

3. Add `HOMEBREW_TAP_TOKEN` env var to the goreleaser-action step in `.github/workflows/release.yml`:
```yaml
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
```
Add a comment on the HOMEBREW_TAP_TOKEN line: `# PAT with Contents:write on homebrew-kinder repo`

Do NOT change any other part of release.yml — only add the one env line.

  <action>Claude cannot create GitHub Personal Access Tokens — this requires human authentication in the browser.</action>
  <instructions>
Create a fine-grained PAT and store it as a secret:

1. Go to https://github.com/settings/personal-access-tokens/new
2. Token name: `HOMEBREW_TAP_TOKEN`
3. Expiration: 90 days (or custom)
4. Resource owner: `PatrykQuantumNomad`
5. Repository access: "Only select repositories" -> select `homebrew-kinder`
6. Repository permissions: Contents -> "Read and write"
7. Click "Generate token" and copy the value

8. Go to https://github.com/PatrykQuantumNomad/kinder/settings/secrets/actions
9. Click "New repository secret"
10. Name: `HOMEBREW_TAP_TOKEN`
11. Value: paste the token from step 7
12. Click "Add secret"
  </instructions>
