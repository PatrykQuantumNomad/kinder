1. In `kinder-site/src/components/InstallCommand.astro`, add a responsive media query to the existing `<style>` block. Append the following CSS after the existing `.copy-btn:hover` rule but before the closing `</style>` tag:

```css
@media (max-width: 50rem) {
  .install-command {
    flex-direction: column;
    align-items: stretch;
    max-width: 100%;
  }

  .copy-btn {
    width: 100%;
    padding: 0.5rem 0.625rem;
  }
}
```

This stacks the command text and copy button vertically on mobile viewports (below Starlight's 50rem breakpoint), prevents horizontal overflow, and makes the copy button full-width for easier tapping. Do NOT add `overflow-x: hidden` to body/html -- fix the source element instead.

2. Create `kinder-site/public/robots.txt` with this exact content:

```
User-agent: *
Allow: /

Sitemap: https://kinder.patrykgolabek.dev/sitemap-index.xml
```

This passes Lighthouse's robots.txt validity audit and points crawlers to Starlight's auto-generated sitemap. Do NOT install `astro-robots-txt` or any npm package -- a static file in `public/` is sufficient.

3. Verify the site builds with zero errors:

```bash
cd kinder-site && npm run build
```

The build should produce the same page count (10 pages) and include `robots.txt` in the `dist/` output.
