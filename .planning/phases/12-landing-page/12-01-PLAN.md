  <name>Task 1: Create InstallCommand and Comparison Astro components</name>
  <files>kinder-site/src/components/InstallCommand.astro, kinder-site/src/components/Comparison.astro</files>
  <action>
Create two custom Astro components in `kinder-site/src/components/`.

**InstallCommand.astro** — A styled code display with a copy-to-clipboard button:
- Display the full install command as a one-liner: `git clone https://github.com/patrykattc/kinder.git && cd kinder && make install`
- Use a `<div class="install-command not-content">` root element. The `not-content` class prevents Starlight's `.sl-markdown-content` styles from interfering.
- Inside: a `<code class="command-text">` displaying the command text, and a `<button class="copy-btn">` labeled "Copy".
- Store the command string in the component frontmatter as a const, pass it to the button via `data-command` attribute.
- Add a client-side `<script>` block (Astro bundles scripts automatically) that attaches a click handler to `.copy-btn`: calls `navigator.clipboard.writeText(command)`, changes button text to "Copied!" for 2 seconds, then reverts to "Copy".
- Scoped `<style>` block: flex layout, `var(--sl-color-black)` background, `1px solid var(--sl-color-gray-5)` border, `0.375rem` border-radius, `0.75rem 1rem` padding, `1.5rem` block margin, `max-width: 42rem` to center-constrain. Command text: `var(--sl-font-mono)` font, `var(--sl-text-sm)` size, `var(--sl-color-accent-high)` color, `overflow-x: auto`, `white-space: nowrap`. Copy button: `var(--sl-color-accent)` background, `var(--sl-color-black)` text, no border, `0.25rem` radius, `font-weight: 600`, hover state with `var(--sl-color-accent-high)`.
- Do NOT use any external clipboard library — `navigator.clipboard.writeText()` is sufficient.

**Comparison.astro** — A two-column before/after comparison (kind vs kinder):
- Root element: `<div class="comparison not-content">`. The `not-content` class is critical to prevent Starlight markdown styles.
- Two columns, each a `<div class="column">`:
  - Left column (kind): `<h3>kind</h3>` + `<ul>` with items: "Bare Kubernetes cluster", "No LoadBalancer support", "No ingress controller", "No metrics (kubectl top fails)", "Default CoreDNS config", "No dashboard"
  - Right column (kinder): `<div class="column kinder">` + `<h3>kinder</h3>` + `<ul>` with items: "Kubernetes cluster", "MetalLB — real LoadBalancer IPs", "Envoy Gateway — Gateway API routing", "Metrics Server — kubectl top and HPA", "CoreDNS — autopath, pods verified, 60s cache", "Headlamp — web dashboard"
- Scoped `<style>`: CSS Grid with `grid-template-columns: 1fr 1fr`, `1.5rem` gap, `2rem` block margin. Column: `1px solid var(--sl-color-gray-5)` border, `0.5rem` radius, `1.25rem` padding. `.column.kinder` gets `border-color: var(--sl-color-accent)`. Headings: `margin-top: 0`, `var(--sl-color-white)` color; `.kinder h3` gets `var(--sl-color-accent)` color. List: no extra margin, `1.25rem` inline-start padding, `var(--sl-color-gray-2)` color.
- Responsive: `@media (max-width: 50rem)` switches to `grid-template-columns: 1fr` (single column stack).
