   `### HPA shows unknown/50%` ‚Äî Symptom: `kubectl describe hpa` shows `<unknown>/50%` for CPU metric. Cause: the target deployment's containers do not have `resources.requests.cpu` set ‚Äî Metrics Server calculates utilization as `current / requested`, and without a request the denominator is undefined. Fix: add `resources.requests.cpu` to the container spec and redeploy.
