CRITICAL fork safety (REL-06 — this is a blocker/concern from STATE.md):
- `project_name: kinder` — explicit, not inferred from go.mod which says `sigs.k8s.io/kind`
- `gomod: proxy: false` — prevents GoReleaser from resolving upstream kind via Go module proxy

Build matrix (REL-01):
- `goos: [linux, darwin, windows]`
- `goarch: [amd64, arm64]`
- `ignore: [{goos: windows, goarch: arm64}]` — 5 platform combinations total
- `env: [CGO_ENABLED=0]`
- `main: .` (repo root has main.go)
- `binary: kinder`

Ldflags (REL-04) — must match Makefile's KIND_VERSION_PKG path exactly:
- `-trimpath`
- `-buildid=`
- `-w`
- `-X=sigs.k8s.io/kind/pkg/internal/kindversion.gitCommit={{ .FullCommit }}`

NOTE: The Makefile also injects `gitCommitCount` via `git describe --tags`. GoReleaser has no built-in template for this. For tagged releases, `versionPreRelease` is `""` so commitCount is unused. For snapshots, GoReleaser's own snapshot suffix handles dev identification. Omit `gitCommitCount` from ldflags — it is safe to skip per research analysis.

Archives (locked decisions):
- `name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"` — lowercase OS per locked naming convention
- `formats: [tar.gz]` with `format_overrides: [{goos: windows, formats: [zip]}]`
- `files: [none*]` — binary-only, no LICENSE/README bundled (locked decision)

Checksum (REL-02):
- `name_template: "checksums.txt"`
- `algorithm: sha256`

Changelog (REL-03 — locked decision: categorized by commit prefix):
- `sort: asc`
- `use: git` (Claude's discretion: use git over github for simplicity, no extra API call)
- Groups: Features (feat:), Bug Fixes (fix:), Documentation (docs:), Other Changes (catchall)
- Filters exclude: chore:, test:, ci:, merge commits

Release section:
- `github: {owner: PatrykQuantumNomad, name: kinder}`
- `draft: false`
- `mode: replace` — allows re-running on same tag if workflow fails (pitfall 3 from research)
- `footer:` with install snippet (locked decision: curl one-liner for quick setup). Use the template from 35-RESEARCH.md's code examples section.

Before hooks:
- `go mod tidy`

Build reproducibility:
- `mod_timestamp: "{{ .CommitTimestamp }}"`

Do NOT include a `brews:` or `homebrew_casks:` section — Homebrew is Phase 36. The `brews:` key is deprecated since GoReleaser v2.10 (per research).

Add two new Makefile targets after the existing `shellcheck` target and before the `.PHONY` line:

```makefile
################################################################################
# =============================== GoReleaser ==================================
# validate goreleaser config — run before any PR touching .goreleaser.yaml

	goreleaser check

	goreleaser build --snapshot --clean
```
