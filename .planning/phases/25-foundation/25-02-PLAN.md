Replace the entire contents of `hack/tools/.golangci.yml` with the v2 format config. The migration mapping from research:

```yaml
version: "2"

run:
  timeout: 3m

linters:
  default: none
  enable:
    - errcheck
    - govet
    - ineffassign
    - staticcheck
    - typecheck
    - gochecknoinits
    - revive
    - misspell
    - unparam
  settings:
    staticcheck:
      checks:
        - all
  exclusions:
    rules:
      # this requires renaming all unused parameters, we'd rather leave a preferred
      # placeholder name in place when implementing an interface etc.
      # we can revisit this later, right now it's generating a lot of new warnings
      # after upgrading golangci-lint
      - text: "^unused-parameter: .*"
        linters:
          - revive

formatters:
  enable:
    - gofmt
```

Key changes from v1 to v2:
- Added `version: "2"` at top (REQUIRED for v2)
- `linters.disable-all: true` -> `linters.default: none`
- Removed `gosimple` from enable list (merged into staticcheck in v2)
- Removed `exportloopref` from enable list (deprecated/removed in v2 — it causes a build error)
- Moved `gofmt` from `linters.enable` to `formatters.enable` (formatting linters are separate in v2)
- Moved `linters-settings` to `linters.settings` (nested under linters)
- Moved `issues.exclude-rules` to `linters.exclusions.rules` (nested under linters.exclusions)
- Preserved the comment explaining the revive unused-parameter exclusion

After writing the config, run `golangci-lint run ./...` from repo root using the built binary. If there are new lint violations from v2, fix them. Common v2-specific new findings:
- intrange linter may flag `for i := 0; i < len(x); i++` patterns (but it's not enabled, so unlikely)
- staticcheck may find new issues since gosimple is now merged in (but it was already enabled)

The lint pass should be clean since we're only enabling linters that were already enabled in v1 (minus the removed ones).

IMPORTANT: Run the lint check on the main module AND the kindnetd module (as lint.sh does both).
