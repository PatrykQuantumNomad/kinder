  <action>
    Create `kinder-site/src/content/docs/known-issues.md` with the following structure:

    **Frontmatter:**
    ```yaml
    ---
    title: Known Issues
    description: Every diagnostic check kinder doctor runs, what it detects, why it matters, and how to fix it.
    ---
    ```

    **Content structure (all sections required):**

    1. **Intro paragraph**: Brief explanation that `kinder doctor` runs 18 diagnostic checks across 8 categories. Link to the [Troubleshooting](/cli-reference/troubleshooting/) page for exit codes and CI usage.

    2. **How to run diagnostics**: Show `kinder doctor` command, `--output json` flag, and exit code table (0=ok, 1=fail, 2=warn).

    3. **One H2 section per category, one H3 per check.** For each check, document:
       - **What it detects** (1-2 sentences)
       - **Why it matters** (what breaks if ignored)
       - **Platforms** (Linux only, or all platforms)
       - **How to fix** (specific commands or configuration changes)

    4. **Categories and checks to document (in this order):**

       **## Runtime**
       - ### Container Runtime -- docker/podman/nerdctl availability and daemon health

       **## Docker**
       - ### Disk Space -- <5GB warn, <2GB fail, `docker system prune` to reclaim
       - ### Daemon JSON Init Config -- `"init": true` causes kind failures, remove or set to false
       - ### Docker Snap -- snap installs have TMPDIR issues, recommend native apt install
       - ### Docker Socket Permissions -- Linux socket permission denied, `sudo usermod -aG docker $USER`

       **## Tools**
       - ### kubectl -- binary not found, install link
       - ### kubectl Version Skew -- >1 minor version difference, kubectl docs link

       **## GPU**
       - ### NVIDIA Driver -- nvidia-smi not found or driver not loaded
       - ### NVIDIA Container Toolkit -- nvidia-container-cli not available
       - ### NVIDIA Docker Runtime -- NVIDIA runtime not configured in Docker

       **## Kernel** (Linux only)
       - ### Inotify Limits -- max_user_watches <524288 or max_user_instances <512, sysctl fix commands
       - ### Kernel Version -- <4.6 is hard blocker (no cgroup namespace), upgrade kernel

       **## Security** (Linux only)
       - ### AppArmor -- profiles interfering with kind containers
       - ### SELinux -- enforcing mode on Fedora, suggest permissive or targeted policy

       **## Platform**
       - ### Firewalld -- nftables backend on Fedora 32+, suggest iptables backend
       - ### WSL2 Cgroup -- WSL2 detected but cgroup v2 not configured, .wslconfig fix
       - ### Rootfs Device -- BTRFS storage driver causes device node issues

       **## Network**
       - ### Subnet Clashes -- Docker network overlaps host routes (VPN, corporate network), configure custom Docker address pools in daemon.json

    5. **## Automatic Mitigations**: Explain that `kinder create cluster` runs `ApplySafeMitigations()` before provisioning. Document what tier-1 means: only env vars and cluster config adjustments, never sudo or system file modification. Note that currently no tier-1 mitigations exist -- the infrastructure is wired and ready for future use.

    **Style guidelines:**
    - Use Starlight `:::note` and `:::tip` admonitions sparingly (for important caveats)
    - Use code blocks for commands: `kinder doctor`, sysctl commands, daemon.json examples
    - Keep each check section concise (5-10 lines max)
    - Use consistent structure: What / Why / Fix format for every check
    - Do NOT use emojis
    - Reference kind's known issues page where applicable: https://kind.sigs.k8s.io/docs/user/known-issues/
