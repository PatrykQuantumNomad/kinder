  <name>Docker network subnet clash detection</name>
  <files>pkg/internal/doctor/subnet.go, pkg/internal/doctor/subnet_test.go, pkg/internal/doctor/check.go</files>
  <behavior>
    --- normalizeAbbreviatedCIDR ---
    - "127" -> "127.0.0.0/8", ok=true (1-octet macOS abbreviation)
    - "169.254" -> "169.254.0.0/16", ok=true (2-octet macOS abbreviation)
    - "192.168.86" -> "192.168.86.0/24", ok=true (3-octet macOS abbreviation)
    - "10.0.0.0/8" -> "10.0.0.0/8", ok=true (already CIDR, pass through)
    - "192.168.86.1/32" -> "192.168.86.1/32", ok=true (explicit CIDR host route)
    - "192.168.86.1" -> ok=false (4-octet without CIDR = host route, skip)
    - "default" -> ok=false (non-numeric, skip)
    - "link#23" -> ok=false (non-numeric, skip)
    - "" -> ok=false (empty, skip)

    --- subnetClashCheck.Run() ---
    - No docker binary: returns single Result with Status="skip", Message contains "docker not found"
    - Docker available, no Docker networks found: returns single Result with Status="ok"
    - Docker available, no host routes: returns single Result with Status="ok"
    - Docker subnet 172.19.0.0/16 + host route 172.16.0.0/12 (VPN): returns Status="warn" (overlaps)
    - Docker subnet 172.19.0.0/16 + host route 172.19.0.0/16 (self-referential): returns Status="ok" (self-route skipped)
    - Docker subnet 172.17.0.0/16 + host route 10.0.0.0/8: returns Status="ok" (no overlap)
    - Multiple Docker networks checked: both "kind" and "bridge" network subnets inspected

    --- getDockerNetworkSubnets ---
    - Parses docker network inspect Go template output: "172.19.0.0/16" -> netip.Prefix
    - Network not found returns empty slice (not error)
    - Filters to IPv4 only (skips IPv6 fc00::/64 entries)

    --- getHostRoutes ---
    - Linux: parses "ip route show" output, first field is CIDR like "172.17.0.0/16"
    - Linux: skips "default" lines
    - macOS: parses "netstat -rn" output using normalizeAbbreviatedCIDR
    - Returns only IPv4 prefixes

    --- allChecks registry ---
    - newSubnetClashCheck() added as 18th check under "// Category: Network (Phase 41)" comment
    - Check Name() returns "network-subnet"
    - Check Category() returns "Network"
    - Check Platforms() returns nil (all platforms)
  </behavior>
