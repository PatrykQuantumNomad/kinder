    Frontmatter: `title: Configuration Reference`, `description: Complete reference for kinder cluster configuration including all addon fields.`

    Content structure (Diataxis: Reference):
    1. Brief intro: "kinder extends kind's cluster configuration with an `addons` section. All addons are enabled by default."
    2. **Cluster configuration** section: Show the basic config structure:
       ```yaml title="kinder-config.yaml"
       kind: Cluster
       apiVersion: kind.x-k8s.io/v1alpha4
       ```

    4. **Addon fields** section with a Markdown table:

       | Field | Type | Default | Description |
       |-------|------|---------|-------------|
       | `addons.metalLB` | bool | `true` | Install MetalLB for LoadBalancer service support |
       | `addons.envoyGateway` | bool | `true` | Install Envoy Gateway with Gateway API CRDs |
       | `addons.metricsServer` | bool | `true` | Install Metrics Server for `kubectl top` and HPA |
       | `addons.coreDNSTuning` | bool | `true` | Apply CoreDNS autopath and cache tuning |
       | `addons.dashboard` | bool | `true` | Install Headlamp web dashboard |

    5. **Disabling addons** section: Show example config that disables MetalLB and the dashboard:
       ```yaml title="minimal-config.yaml"
       kind: Cluster
       apiVersion: kind.x-k8s.io/v1alpha4
       addons:
         metalLB: false
         dashboard: false
       ```
       Note: Omitted fields default to `true`. You only need to specify fields you want to disable.
    6. **Compatibility with kind** section: Note that kinder's config format extends kind's v1alpha4 format. Existing kind config files work unchanged — the `addons` section is optional. Link to kind's documentation for node configuration, networking, and other standard fields.
