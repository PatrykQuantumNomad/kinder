1. Create `pkg/internal/doctor/clusterskew.go` implementing a `clusterNodeSkewCheck` struct.

2. The check uses dependency injection following the established doctor check pattern. The struct has injectable functions for both node listing and container image retrieval:
   ```go
   type clusterNodeSkewCheck struct {
       lookPath  func(string) (string, error)
       execCmd   func(name string, args ...string) exec.Cmd
       listNodes func() ([]nodeEntry, error)  // abstracts provider.ListNodes + KubeVersion + container inspect
   }

   type nodeEntry struct {
       Name    string
       Role    string
       Version string  // live version from /kind/version (e.g., "v1.31.2")
       Image   string  // container image from runtime inspect (e.g., "kindest/node:v1.31.2")
   }
   ```

3. The `newClusterNodeSkewCheck()` constructor creates a real implementation that:
   a. Injects `lookPath: osexec.LookPath` and `execCmd: exec.Command` (same as all other checks).
   b. Injects a `listNodes` function that:
      - Detects the runtime binary using the same pattern as `runtime.go`: iterate `["docker", "podman", "nerdctl"]` with `lookPath`, use the first one found. Store as `binaryName`.
      - Creates a provider via `cluster.NewProvider()` with a no-op logger.
      - Calls `provider.ListNodes(cluster.DefaultName)` to get `[]nodes.Node`.
      - For each node:
        - Calls `nodeutils.KubeVersion(n)` to get the live version from `/kind/version`.
        - Calls `n.Role()` to get the role.
        - Retrieves the container image using `exec.Command(binaryName, "inspect", "--format", "{{.Config.Image}}", n.String())`. This is the same pattern used by `Role()` in `common/node.go` (which calls `exec.Command(n.BinaryName, "inspect", "--format", ..., n.Name)`). The `binaryName` comes from the runtime detection above. `n.String()` returns the container name.
        - Parses the inspect output to get the full image string (e.g., `"kindest/node:v1.31.2"`).
      - Returns `[]nodeEntry` with all four fields populated.

4. `Run()` logic:
   a. Call `listNodes()`. If error or empty, return a skip result with message "(no cluster found)".
   b. Parse each node's live Version with `version.ParseSemantic`.
   c. Find control-plane nodes and their minor version.
   d. Check HA control-plane same-minor-version requirement. If mismatched, emit a warn result.
   e. Check workers within 3 minor versions of control-plane. Collect violations.
   f. **Config drift detection:** For each node, extract the version tag from the Image string using `imageTagVersion(entry.Image)` (same helper logic as in validate.go -- extract substring after last `:`; strip `@sha256:` digest suffix first). Compare `imageTagVersion(entry.Image)` against `entry.Version`. If they differ, it means the container was created with one image version but is now reporting a different live version -- this is config drift. Collect drift entries.
   g. If skew violations found, return `warn` with:
      - Same table format as create-time validation (per user decision):
        ```
        Node        Role           Version  Delta
        worker-2    worker         v1.27.1  -4
        ```
      - Reason: "cluster node version-skew exceeds Kubernetes policy (max 3 minor versions)"
      - Fix: "update <node> to v1.28+ ..." (remediation hints matching create-time format)
   h. If drift detected (independently of skew), return an additional `warn` result:
      - Message: "config drift detected: <node> image is <imageVersion> but running <liveVersion>"
      - Reason: "container image version does not match live kubelet version"
      - Fix: "recreate the node with the intended image, or investigate why /kind/version differs from the container image"
   i. If no violations and no drift, return `ok` with message listing all node versions.
   j. Severity is `warn` (per Claude's Discretion decision, consistent with kubectlVersionSkewCheck).

5. Add an unexported helper `imageTagVersion(image string) (string, error)` in `clusterskew.go` that extracts the version tag from a container image string. Strip any `@sha256:` digest suffix first, then extract the substring after the last `:`. Return error if no tag found.

   - Inject `listNodes` returning a `nodeEntry` where `Image = "kindest/node:v1.31.2"` and `Version = "v1.31.2"` (no drift) -> expect no drift warning.
