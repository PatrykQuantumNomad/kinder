2. **## Prerequisites** -- Bullet list:
   - kinder installed (link to `/installation`)
   - Docker (or Podman) installed and running
   - `kubectl` installed and on PATH
   - A text editor

3. **## Step 1: Create the cluster** -- `kinder create cluster` command. Add `:::tip` noting the local registry is enabled by default and accessible at `localhost:5001`.

4. **## Step 2: Verify the registry is running** -- `docker ps --filter name=kind-registry` with expected output showing `registry:2` container. Then verify push access: explain this confirms the registry is ready.

5. **## Step 3: Create a simple application** -- Create two files in a new directory:

   `main.go`:
   ```go
   package main

   import (
       "fmt"
       "net/http"
   )

   func main() {
       http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
           fmt.Fprintln(w, "Hello from v1!")
       })
       http.ListenAndServe(":8080", nil)
   }
   ```

   `Dockerfile`:
   ```dockerfile
   FROM golang:1.23-alpine AS builder
   WORKDIR /app
   COPY main.go .
   RUN go build -o server main.go

   FROM alpine:latest
   COPY --from=builder /app/server /server
   EXPOSE 8080
   CMD ["/server"]
   ```

   Add a `:::tip` that this Go server is just an example -- the same workflow works with any language/framework. The key is Docker + localhost:5001.

6. **## Step 4: Build and push v1** -- `docker build -t localhost:5001/myapp:v1 .` then `docker push localhost:5001/myapp:v1`. Show expected push output with digest.

7. **## Step 5: Deploy to the cluster** -- Full Deployment + Service YAML. Deployment uses `image: localhost:5001/myapp:v1`. Service type ClusterIP, port 80 targeting 8080. Apply and show `kubectl get pods` with expected output.

8. **## Step 6: Access the application** -- `kubectl port-forward svc/myapp 8080:80` then `curl http://localhost:8080`. Expected output: `Hello from v1!`. Add a `:::note` that port-forward works on all platforms (Linux, macOS, Windows).

9. **## Step 7: Make a change and iterate** -- Edit `main.go` to change "v1" to "v2". Then the iteration loop:
   ```sh
   docker build -t localhost:5001/myapp:v2 .
   docker push localhost:5001/myapp:v2
   kubectl set image deployment/myapp myapp=localhost:5001/myapp:v2
   kubectl rollout status deployment/myapp
   ```
   Show expected rollout output: `deployment "myapp" successfully rolled out`.

   Add a `:::tip[Iteration pattern]` callout summarizing the loop: edit code -> `docker build` -> `docker push` -> `kubectl set image` -> verify. Use a new tag for each iteration (`:v2`, `:v3`, etc.).

10. **## Step 8: Verify the new version** -- Re-run `kubectl port-forward svc/myapp 8080:80` (the previous one was terminated) and `curl http://localhost:8080`. Expected output: `Hello from v2!`.

11. **## Clean up** -- `kinder delete cluster` and optionally remove the project directory.
