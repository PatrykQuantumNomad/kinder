**Imports to add:** `bytes`, `fmt`, `strings` from stdlib; `sigs.k8s.io/kind/pkg/cluster/nodeutils`, `sigs.k8s.io/kind/pkg/errors` from project. Remove the current lone `actions` import and include it alongside the new ones. Follow the project import convention: stdlib block, blank line, project imports block.

**Execute function — 6 steps:**

1. **Status + control plane node:** Start status with `"Tuning CoreDNS"` (not the stub's `"Tuning CoreDNS (stub)"`). Get allNodes via `ctx.Nodes()`, find controlPlanes via `nodeutils.ControlPlaneNodes(allNodes)`, assert `len(controlPlanes) > 0`, assign `node := controlPlanes[0]`. Follow the exact pattern from `installmetricsserver/metricsserver.go`.

2. **Read current Corefile:** Declare `var corefile bytes.Buffer`. Run `node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "get", "configmap", "coredns", "--namespace=kube-system", "-o", "jsonpath={.data.Corefile}").SetStdout(&corefile).Run()`. This is the `installcni` pattern (bytes.Buffer + SetStdout). Convert to string: `corefileStr := corefile.String()`.

3. **Guard checks:** Verify the Corefile contains expected tokens BEFORE modifying. Three `strings.Contains` checks:
   - `"pods insecure"` — if missing, return `errors.New("CoreDNS Corefile does not contain 'pods insecure'; cannot patch safely")`
   - `"cache 30"` — if missing, return `errors.New("CoreDNS Corefile does not contain 'cache 30'; cannot patch safely")`
   - `"kubernetes cluster.local"` — if missing, return `errors.New("CoreDNS Corefile does not contain 'kubernetes cluster.local'; cannot patch safely")`
   These guards satisfy DNS-04 (safe patching) by failing early if the Corefile format is unexpected.

4. **Three string transforms:**
   - DNS-02: `strings.ReplaceAll(corefileStr, "pods insecure", "pods verified")` — required prerequisite for autopath
   - DNS-01: `strings.ReplaceAll(corefileStr, "    kubernetes cluster.local", "    autopath @kubernetes\n    kubernetes cluster.local")` — inserts autopath as a sibling directive BEFORE the kubernetes block (not inside it). The 4-space indent matches the Corefile's server block indentation.
   - DNS-03: `strings.ReplaceAll(corefileStr, "cache 30", "cache 60")` — increases cache TTL

5. **Write-back via ConfigMap YAML envelope:** Use `fmt.Sprintf(configMapTemplate, indentCorefile(corefileStr))` to build the YAML. Pipe to `kubectl apply -f -` via `node.Command(...).SetStdin(strings.NewReader(configMapYAML)).Run()`. This is the `installmetallb` pattern (strings.NewReader + SetStdin).

6. **Rollout restart + wait:** Run `kubectl rollout restart --namespace=kube-system deployment/coredns` then `kubectl rollout status --namespace=kube-system deployment/coredns --timeout=60s`. This satisfies DNS-04 (rolling restart maintains at least one live CoreDNS pod throughout).

**Package-level const and helper:**

Add `configMapTemplate` as a package-level const (the YAML envelope for write-back):
```
apiVersion: v1
kind: ConfigMap
metadata:
  name: coredns
  namespace: kube-system
data:
  Corefile: |
%s
```

Add `indentCorefile(corefile string) string` as an unexported helper that splits on `"\n"`, prepends `"    "` (4 spaces) to each non-empty line, and joins back with `"\n"`. Empty lines are left as empty strings (avoids trailing whitespace — see Pitfall 5 in research).

**What NOT to do and why:**
- Do NOT use `kubectl patch` with JSON — multi-line Corefile requires JSON escaping of all newlines. The YAML envelope approach is the established project pattern.
- Do NOT hardcode a replacement Corefile — violates DNS-01 ("patched, not replaced"). Only modify the three specific directives.
- Do NOT rely on the `reload` plugin for config propagation — ConfigMap volume propagation takes 60-90s. Rollout restart is 10-20s and deterministic.
- Do NOT add a `manifests/` subdirectory — unlike MetalLB/Metrics Server, CoreDNS tuning modifies an existing ConfigMap, not a new manifest.
- Do NOT create an `//go:embed` directive — there is nothing to embed.
