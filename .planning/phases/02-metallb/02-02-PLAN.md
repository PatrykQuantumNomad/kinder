  <files>pkg/cluster/internal/create/actions/installmetallb/metallb.go</files>
  <action>
    Replace the stub Execute function in metallb.go with the full MetalLB installation logic. The file must retain the Apache 2.0 license header and package declaration.

    Add imports:
    - `_ "embed"` (blank import for go:embed)
    - `"fmt"`, `"strings"`
    - `"sigs.k8s.io/kind/pkg/errors"`
    - `"sigs.k8s.io/kind/pkg/cluster/nodeutils"`

    Add go:embed directive above a package-level var:
    ```go
    //go:embed manifests/metallb-native.yaml
    var metalLBManifest string
    ```

    Add CR template constant:
    ```go
    const crTemplate = `---
    apiVersion: metallb.io/v1beta1
    kind: IPAddressPool
    metadata:
      name: kind-pool
      namespace: metallb-system
    spec:
      addresses:
      - %s
    ---
    apiVersion: metallb.io/v1beta1
    kind: L2Advertisement
    metadata:
      name: kind-l2advert
      namespace: metallb-system
    spec:
      ipAddressPools:
      - kind-pool
    `
    ```

    Implement Execute with these sequential steps:

    1. `ctx.Status.Start("Installing MetalLB")` with `defer ctx.Status.End(false)`

    2. Get control plane node:
       ```go
       allNodes, err := ctx.Nodes()
       controlPlanes, err := nodeutils.ControlPlaneNodes(allNodes)
       node := controlPlanes[0]
       ```

    3. Check for rootless Podman and warn:
       ```go
       info, err := ctx.Provider.Info()
       if info.Rootless {
           ctx.Logger.Warn("MetalLB L2 speaker cannot send ARP in rootless mode; LoadBalancer services may not receive EXTERNAL-IP")
       }
       ```

    4. Detect provider name for subnet detection. Since Provider interface lacks String(), use fmt.Stringer type assertion:
       ```go
       providerName := "docker" // default fallback
       if s, ok := ctx.Provider.(fmt.Stringer); ok {
           providerName = s.String()
       }
       ```

    5. Apply the embedded MetalLB manifest via kubectl:
       ```go
       node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "apply", "-f", "-").
           SetStdin(strings.NewReader(metalLBManifest)).Run()
       ```
       Wrap error: "failed to apply MetalLB manifest"

    6. Wait for controller webhook readiness:
       ```go
       node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf",
           "wait", "--namespace=metallb-system",
           "--for=condition=Available", "deployment/controller",
           "--timeout=120s").Run()
       ```
       Wrap error: "MetalLB controller did not become available"

    7. Detect subnet from host network:
       ```go
       subnet, err := detectSubnet(providerName)
       ```
       Wrap error: "failed to detect kind network subnet for MetalLB"

    8. Carve pool range:
       ```go
       poolRange, err := carvePoolFromSubnet(subnet)
       ```
       Wrap error: "failed to compute MetalLB IP pool"

    9. Apply IPAddressPool + L2Advertisement CRs:
       ```go
       crYAML := fmt.Sprintf(crTemplate, poolRange)
       node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "apply", "-f", "-").
           SetStdin(strings.NewReader(crYAML)).Run()
       ```
       Wrap error: "failed to apply MetalLB CRs"

    10. `ctx.Status.End(true)` and `return nil`

    Use `errors.Wrap(err, "message")` for all error wrapping (not fmt.Errorf).
    Use `"sigs.k8s.io/kind/pkg/errors"` (not standard library errors).

    The crTemplate YAML must be properly indented -- the `addresses:` list item uses `- %s` where %s is the pool range like "172.18.255.200-172.18.255.250".

    MLB-08 (disable via config) is already handled by the runAddon gating in create.go from Phase 1 -- the Execute function only runs when MetalLB is enabled.
