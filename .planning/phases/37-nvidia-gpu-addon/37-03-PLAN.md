    8. **Troubleshooting** (critical for SITE-02):

       ### Pod stuck in Pending with "0/1 nodes are available: insufficient nvidia.com/gpu"

       **Symptom:** Pod requesting `nvidia.com/gpu` stays `Pending`.

       **Cause:** The NVIDIA device plugin has not registered any GPUs. Three common reasons:
       - (a) nvidia-container-toolkit is not installed or Docker is not configured with the nvidia runtime
       - (b) The host GPU is not visible inside the kind node container
       - (c) The device plugin pod is crashing

       **Fix:**
       ```sh
       # Check device plugin pod status
       kubectl get pods -n kube-system -l name=nvidia-device-plugin-ds

       # Check if GPU is visible inside the node
       docker exec -it <cluster-name>-control-plane nvidia-smi

       # If nvidia-smi fails inside the container, reconfigure Docker:
       sudo nvidia-ctk runtime configure --runtime=docker
       sudo systemctl restart docker
       # Then recreate the cluster
       ```

       ### kinder create cluster fails with "nvidia-container-toolkit not found"

       **Symptom:** `kinder create cluster` with `nvidiaGPU: true` fails immediately.

       **Cause:** The `nvidia-ctk` binary is not on your PATH.

       **Fix:**
       ```sh
       sudo apt-get install -y nvidia-container-toolkit
       sudo nvidia-ctk runtime configure --runtime=docker
       sudo systemctl restart docker
       ```

       ### GPU addon skipped with "Linux only" message

       **Symptom:** Cluster creates successfully but GPU addon shows "skipping on darwin (Linux only)".

       **Cause:** The NVIDIA GPU addon only works on Linux hosts. macOS and Windows do not support GPU passthrough to Docker containers.

       **Fix:** No fix — this is expected behavior. Use a Linux host or Linux VM with GPU passthrough.
