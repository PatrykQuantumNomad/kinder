  <behavior>
    WSL2 detection (multi-signal):
    - Test: /proc/version contains "microsoft" + WSL_DISTRO_NAME set + cgroup controllers present -> status "ok" with controller list
    - Test: /proc/version contains "Microsoft" (case variant) + WSL_DISTRO_NAME set + cgroup controllers present -> status "ok"
    - Test: /proc/version contains "microsoft" + WSLInterop file exists + cgroup controllers present -> status "ok"
    - Test: /proc/version contains "microsoft" + WSLInterop-late file exists + cgroup controllers present -> status "ok" (WSL 4.1.4+)
    - Test: /proc/version contains "microsoft" but NO second signal (Azure VM) -> status "ok", message "Not running under WSL2"
    - Test: /proc/version does NOT contain "microsoft" -> status "ok", message "Not running under WSL2"
    - Test: /proc/version unreadable -> status "ok", message "Not running under WSL2"

    Cgroup v2 controller check (when WSL2 confirmed):
    - Test: WSL2 confirmed + cgroup.controllers readable with "cpu memory pids io" -> status "ok"
    - Test: WSL2 confirmed + cgroup.controllers unreadable -> status "warn" with cgroup v2 fix link
    - Test: WSL2 confirmed + cgroup.controllers missing "cpu" -> status "warn" listing missing controllers
    - Test: WSL2 confirmed + cgroup.controllers missing "memory pids" -> status "warn" listing missing controllers

    Metadata:
    - Test: Name()="wsl2-cgroup", Category()="Platform", Platforms()=["linux"]
  </behavior>
  <action>
    Write wsl2_test.go first with table-driven tests. The test struct should include fields for:
    - files map[string]string (content for readFile)
    - fileErrors map[string]error (errors for specific readFile paths, for fine-grained control)
    - env map[string]string (for getenv)
    - statResults map[string]bool (true = file exists for stat)
    - wantStatus, wantMsgContain

    Then implement wsl2.go:
    - wsl2Check struct with three deps:
      - readFile func(string) ([]byte, error)
      - getenv func(string) string
      - stat func(string) (os.FileInfo, error)
    - Constructor newWSL2Check() with os.ReadFile, os.Getenv, os.Stat defaults
    - Name() = "wsl2-cgroup", Category() = "Platform", Platforms() = []string{"linux"}
    - isWSL2() bool method implementing multi-signal detection:
      1. Read /proc/version, check for "microsoft" (case-insensitive via strings.ToLower)
      2. If no "microsoft" -> false
      3. Signal 2a: getenv("WSL_DISTRO_NAME") != "" -> true
      4. Signal 2b: stat("/proc/sys/fs/binfmt_misc/WSLInterop") no error -> true
      5. Signal 2c: stat("/proc/sys/fs/binfmt_misc/WSLInterop-late") no error -> true (WSL 4.1.4+)
      6. No second signal -> false (Azure VM case)
    - Run() method:
      1. If !isWSL2() -> ok "Not running under WSL2"
      2. Read /sys/fs/cgroup/cgroup.controllers
      3. If error -> warn "cgroup v2 controllers not available" with fix link to spurin/wsl-cgroupsv2
      4. Parse controllers string, check for required: "cpu", "memory", "pids"
      5. If missing -> warn listing missing controllers with fix link
      6. All present -> ok with controller list

    CRITICAL for Azure VM false positive prevention: isWSL2() MUST require two signals. The /proc/version "microsoft" check alone is NOT sufficient.

    Use Apache 2.0 license header.
