<objective>
Implement the capture-side of snapshot creation: read etcd state, container images, local-path PV contents, topology fingerprint, addon versions, and reconstruct a minimal kind-config YAML from a running kinder cluster. Each capture writes its raw bytes to a host temp file and returns its sha256 hex digest, suitable for feeding directly into Plan 01's `bundle.WriteBundle`.

Purpose: Deliver the SC1 capture surface ("etcd state, all loaded container images, local-path-provisioner PV contents") and the LIFE-08 metadata sources (k8s version, addon versions, image-bundle digest, topology). This plan does NOT orchestrate pause/resume — that is Plan 04's job. Each function takes already-resolved nodes (a `nodes.Node` for the bootstrap CP, and `[]nodes.Node` for workers), so Plan 04 can sequence them around `lifecycle.Pause`.

Output:
- `etcd.go` — etcdctl snapshot save via `crictl exec`, then copy to host.
- `images.go` — `ctr --namespace=k8s.io images export` from CP node.
- `pvs.go` — tar `/opt/local-path-provisioner` from each node that has it.
- `topology.go` — ClassifyFn-injected topology + kubectl-based addon version discovery.
- `kindconfig.go` — reconstruct minimal v1alpha4 YAML from observed state.
- Unit tests using FakeNode/FakeCmd patterns established in v1.4.
</objective>
