    Implement the tar.gz bundle layer. The writer streams entries through `gzip.NewWriter -> tar.NewWriter`, with `io.MultiWriter(file, sha256.New())` capturing the gzip stream's hash in a single pass (per RESEARCH.md §7).

    1. `bundle.go` API:
       ```go
       var BundleEntries = []string{
           EntryEtcd, EntryImages, EntryPVs, EntryConfig, EntryMetadata,
       }

       // Component represents one tar entry sourced from a file on disk so the
       // tar header Size is known up front (avoids buffering GBs in memory).
       type Component struct {
           Name string // one of EntryEtcd | EntryImages | EntryPVs | EntryConfig
           Path string // host filesystem path containing the raw component bytes
       }

       // WriteBundle writes <destPath> (tar.gz) plus <destPath>.sha256 sidecar.
       // Components are written in BundleEntries order; metadata is appended last
       // (it carries archiveDigest = sha256 of everything BEFORE metadata is written).
       // ACTUALLY: we cannot include archiveDigest in metadata.json that we then
       // tar — it's recursive. Resolution: ArchiveDigest is computed over the
       // FINAL .tar.gz file, written ONLY to the sidecar .sha256 file and to
       // metadata.json IF the caller chooses (planning note: leave empty in the
       // tarred metadata.json; populate ArchiveDigest in the returned *Metadata
       // for the in-memory store cache and for SnapshotStore.List to read from
       // the sidecar — single source of truth = sidecar).
       func WriteBundle(ctx context.Context, destPath string, comps []Component, meta *Metadata) (archiveDigest string, err error)

       // OpenBundle returns a reader that streams entries by name. Caller must
       // Close. Used by Restore (Plan 03) to extract etcd.snap / images.tar /
       // pvs.tar to temp files before invoking node operations.
       type BundleReader interface {
           Metadata() *Metadata
           Open(entryName string) (io.ReadCloser, error)
           Close() error
       }
       func OpenBundle(path string) (BundleReader, error)

       // VerifyBundle re-hashes the archive and compares with the sidecar.
       // Returns ErrCorruptArchive on mismatch.
       var ErrCorruptArchive = errors.New("snapshot archive corrupted: sha256 mismatch")
       func VerifyBundle(path string) error
       ```

    2. Implementation notes:
       - Use `archive/tar` + `compress/gzip` + `crypto/sha256` from stdlib only.
       - For each component: `os.Stat` to learn `Size`, then `io.Copy(tarWriter, file)`.
       - Metadata entry: marshal `meta` (with ArchiveDigest left "" inside the tar — correct, because computing it requires the full file written first).
       - After `gz.Close()`, close the file, then read it back to compute final sha256 for the sidecar — OR keep the streaming hasher: `mw := io.MultiWriter(f, h)` where `h` hashes everything written. After Close(), `archiveDigest = hex.EncodeToString(h.Sum(nil))`.
       - Sidecar format (sha256sum convention):
         `<hex-of-archive>  <basename>.tar.gz\n`
       - Mode for both files: 0600. Parent dir: caller's responsibility (store.go).

    1. `store.go`:
       ```go
       // DefaultRoot returns ~/.kinder/snapshots — used when caller passes "".
       func DefaultRoot() (string, error) // os.UserHomeDir + filepath.Join

       type Info struct {
           Name        string    // snap-20260506-120000 OR user-supplied
           ClusterName string
           Path        string    // absolute path to .tar.gz
           Size        int64     // sum of .tar.gz + .sha256
           CreatedAt   time.Time // mtime of .tar.gz
           Metadata    *Metadata // nil if metadata.json could not be parsed
           Status      string    // "ok" | "corrupt" | "unknown"
       }

       type SnapshotStore struct {
           root        string // <root>/<cluster>/
           clusterName string
       }
       func NewStore(root, clusterName string) (*SnapshotStore, error)
           // mkdir -p root with 0700; mkdir <root>/<cluster> with 0700
       func (s *SnapshotStore) List(ctx context.Context) ([]Info, error)
           // glob *.tar.gz, parse name (strip .tar.gz), os.Stat, read .sha256 sidecar,
           // probe metadata.json by opening tar (use OpenBundle(...).Metadata()),
           // run sidecar-only sha256 check (NOT a full re-hash — that's lazy/ondemand)
           // sort newest-first by CreatedAt.
       func (s *SnapshotStore) Open(ctx context.Context, name string) (BundleReader, *Info, error)
           // returns ErrSnapshotNotFound if missing
       func (s *SnapshotStore) Delete(ctx context.Context, name string) error
           // removes <name>.tar.gz and <name>.tar.gz.sha256; ENOENT -> ErrSnapshotNotFound
       func (s *SnapshotStore) Path(name string) string  // <root>/<cluster>/<name>.tar.gz

       var ErrSnapshotNotFound = errors.New("snapshot not found")
       ```
       STATUS computation in List is fast: just compare archive's actual sha256 to the sidecar value via `VerifyBundle`. It IS a full re-hash; on a 5GB archive that takes ~5s. To stay under 1s for `kinder snapshot list`: status defaults to "unknown"; only fully verify when caller explicitly opts in. **Decision (Claude's discretion):** add a flag `func (s *SnapshotStore) ListVerify(ctx, mode)` where mode ∈ {`StatusFast`, `StatusFull`}; default in CLI `list` command is `StatusFast` (status="ok" if sidecar parses + matches archive size implied by sidecar, else "corrupt"; "unknown" if sidecar missing). Document inline; align CLI in Plan 05.

    2. `prune.go` — pure functions over `[]Info`:
       ```go
       type Policy struct {
           KeepLastN  int
           OlderThan  time.Duration  // 0 = no policy
           MaxSize    int64          // 0 = no policy; bytes
       }
       // PrunePlan returns the subset of `infos` to delete to satisfy `policy`.
       // ALL three policies, when set, are AND-combined: a snapshot is deleted if
       // any policy would delete it. infos must be sorted newest-first (caller's
       // responsibility — SnapshotStore.List already does this).
       func PrunePlan(infos []Info, policy Policy, now time.Time) []Info

       // KeepLast: delete everything past the first N items.
       func KeepLast(infos []Info, n int) []Info

       // OlderThan: delete items where now.Sub(item.CreatedAt) > d.
       func OlderThan(infos []Info, d time.Duration, now time.Time) []Info

       // MaxSize: starting from oldest, delete until total Size <= max.
       func MaxSize(infos []Info, max int64) []Info
       ```
