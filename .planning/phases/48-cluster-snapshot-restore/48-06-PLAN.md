<task type="auto">
  <name>Task 2: Restore-refusal cases + STATUS=corrupt integration tests</name>
  <files>
    pkg/internal/snapshot/restore_refusal_integration_test.go
  </files>
  <action>
    Three tests, same `//go:build integration` tag and `integration.MaybeSkip(t)` gate.

    1. `TestIntegrationRestoreRefusesOnK8sMismatch`:
       - Create cluster, take snapshot "v1".
       - Tamper: `docker exec <cp-container> sh -c 'echo v9.99.99 > /kind/version'`.
       - Run `kinder snapshot restore <cluster> v1`.
       - Assert exit code != 0; stderr contains "k8s version mismatch" (or wrap message from ErrCompatK8sMismatch).
       - Cleanup.

    2. `TestIntegrationRestoreRefusesOnTopologyMismatch`:
       - Create a cluster with 1 CP + 2 workers (use a multi-node config file under testdata/).
       - Take snapshot "topo".
       - Tamper: `docker stop <worker-2>` (do NOT use `kinder pause` — we want partial topology, not paused-state).
       - Wait for the cluster API to recognize the worker as NotReady (briefly).
       - Run `kinder snapshot restore <cluster> topo`.
         **Subtle point:** Plan 04's restore CALLS `lifecycle.Pause` AFTER pre-flight, so a stopped-worker scenario actually means: live `provider.ListNodes` returns 3 entries (docker ps -a sees them), but `lifecycle.ClassifyNodes` operates on the entire set regardless of running state. To make this test deterministic, the topology mismatch must be a node-COUNT mismatch — easier to engineer by `docker rm -f <worker-2>` so `provider.ListNodes` returns only 2.
       - Switch tampering to: `docker rm -f <worker-2-container>`.
       - Assert exit code != 0; stderr contains "topology mismatch".
       - Cleanup.

    3. `TestIntegrationRestoreRefusesOnAddonMismatch`:
       - Create cluster with `--local-path` (so local-path-provisioner addon is installed at a known version).
       - Take snapshot "addon-base". Confirm metadata.json captured `addonVersions["localPath"]` to a non-empty string.
       - Tamper: bump the live local-path-provisioner deployment image to a fake version that snapshot.CaptureAddonVersions will read back differently:
         ```
         kubectl -n local-path-storage set image deployment/local-path-provisioner local-path-provisioner=rancher/local-path-provisioner:v0.0.99
         kubectl -n local-path-storage rollout status deployment/local-path-provisioner --timeout=60s
         ```
       - Run `kinder snapshot restore <cluster> addon-base`.
       - Assert exit code != 0; stderr (lowercased) contains the substring "addon" (matches the wrap message of `ErrCompatAddonMismatch` from Plan 04 compat.go).
       - Assert exit code != 0; stderr does NOT contain "k8s version" or "topology" (i.e. only the addon dimension fired — narrows the assertion to the right sentinel).
       - Cleanup.

    4. `TestIntegrationListShowsCorrupt`:
       - Create cluster, take snapshot "victim".
       - Read first 1KB of `~/.kinder/snapshots/<cluster>/victim.tar.gz`, flip byte at offset 512, write back.
       - Delete the sidecar `victim.tar.gz.sha256` would force STATUS="unknown"; we want STATUS="corrupt", so leave the sidecar (it has the OLD/correct sha256).
       - Run `kinder snapshot list <cluster>`.
       - Assert stdout has a row for "victim" with STATUS column == "corrupt".
       - Also assert `kinder snapshot list --output json` returns the same Status field.
       - Cleanup.

    Same helpers as Task 1.
  </action>
  <verify>
