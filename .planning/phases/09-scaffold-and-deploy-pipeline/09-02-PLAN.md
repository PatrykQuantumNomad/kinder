<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure DNS and GitHub Pages</name>
  <action>
    The user must complete three configuration steps that cannot be automated by Claude.
    These should be done in order, with waiting periods between steps.
  </action>
  <instructions>
    **Step 1: Configure DNS CNAME record**

    Go to your DNS provider (registrar for patrykgolabek.dev) and create a CNAME record:
    - Type: CNAME
    - Name: `kinder` (or `kinder.patrykgolabek.dev` depending on your provider's format)
    - Value: `patrykattc.github.io`
    - TTL: 3600 (or provider default)

    **Step 2: Wait for DNS propagation, then verify**

    DNS propagation can take minutes to hours. Check with:
    ```bash
    dig kinder.patrykgolabek.dev CNAME +short
    ```
    Expected output: `patrykattc.github.io.`

    Do NOT proceed to Step 3 until DNS resolves correctly.

    **Step 3: Configure GitHub Pages settings**

    Go to https://github.com/patrykattc/kinder/settings/pages and configure:

    1. **Build and deployment -> Source**: Select "GitHub Actions" (NOT "Deploy from a branch")
    2. **Custom domain**: Enter `kinder.patrykgolabek.dev` and click Save
       - GitHub will verify DNS. If it shows "DNS check successful," proceed.
       - If it shows an error, wait longer for DNS propagation and try again.
    3. **Enforce HTTPS**: Check this box (may be greyed out until GitHub provisions the Let's Encrypt certificate — this can take up to 24 hours after DNS verification)

    **Step 4: Trigger the first deploy**

    Push the kinder-site/ changes from Plan 01 to main (if not already pushed). The deploy-site.yml workflow should trigger automatically. Verify at:
    https://github.com/patrykattc/kinder/actions

    Wait for the workflow to complete (both Build and Deploy jobs should succeed).
  </instructions>
  <resume-signal>Type "configured" when DNS is set, GitHub Pages source is "GitHub Actions", custom domain is entered, and the first deploy workflow has completed successfully. If HTTPS enforcement is still pending (certificate provisioning), note that — we can verify HTTPS separately.</resume-signal>
