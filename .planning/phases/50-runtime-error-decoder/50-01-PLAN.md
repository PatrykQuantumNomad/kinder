<objective>
Build the pure pattern catalog + matcher engine for the runtime error decoder. This plan
delivers the data model (DecodePattern / DecodeMatch / DecodeResult), the >=15-entry
seed catalog covering kubelet, kubeadm, containerd, docker, and addon-startup categories
(DIAG-02 / SC2), and the I/O-free matchLines() function. No live cluster contact, no
CLI surface — just the brain of the decoder so downstream plans can wire collectors,
auto-fix, and the cobra subcommand against a stable in-memory contract.

Purpose: SC2 mandates >=15 patterns shipping at v1; landing the catalog first means
50-02's RunDecode and 50-03's CLI never have to scope-creep their tests with stub
patterns. SC3 mandates four fields per match (pattern, explanation, fix, link) — the
struct shape is fixed here.

Output:
- pkg/internal/doctor/decode.go (types + matcher)
- pkg/internal/doctor/decode_catalog.go (>=15 patterns, all five categories)
- pkg/internal/doctor/decode_test.go + decode_catalog_test.go
</objective>
