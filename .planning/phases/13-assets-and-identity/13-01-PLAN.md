<task type="auto">
  <name>Task 1: Wire favicon, add OG meta tags, and generate OG image</name>
  <files>kinder-site/astro.config.mjs, kinder-site/public/og.png</files>
  <action>
**Step A: Update astro.config.mjs** — Add three config properties to the starlight() call:

1. `favicon: '/favicon.svg'` — points to existing `public/favicon.svg`. Place this after `title: 'kinder'`.

2. `disable404Route: true` — required BEFORE Task 2 creates 404.astro (route collision pitfall). Place this after `favicon`.

3. `head: [...]` — global meta tags array. Place after `disable404Route`. Include these five meta tag objects:
   - `{ tag: 'meta', attrs: { property: 'og:image', content: 'https://kinder.patrykgolabek.dev/og.png' } }`
   - `{ tag: 'meta', attrs: { property: 'og:title', content: 'kinder' } }`
   - `{ tag: 'meta', attrs: { property: 'og:description', content: 'kind, but with everything you actually need.' } }`
   - `{ tag: 'meta', attrs: { name: 'twitter:card', content: 'summary_large_image' } }`
   - `{ tag: 'meta', attrs: { name: 'twitter:image', content: 'https://kinder.patrykgolabek.dev/og.png' } }`

Keep all existing config (sidebar, social, customCss, components) unchanged.

**Step B: Generate public/og.png** — Create a one-off Node.js script (do NOT commit it) that uses sharp (already installed) to generate a 1200x630 PNG:

1. Create an SVG string programmatically (1200x630 viewBox) with:
   - Background: `hsl(220, 15%, 7%)` (matches `--sl-color-black` from theme.css)
   - The kinder snowflake SVG from `public/favicon.svg` centered, rendered at ~200px, filled with `hsl(185, 100%, 45%)` (the accent cyan). Remove the `<style>` block and `prefers-color-scheme` media query from the SVG — hardcode the fill to the accent cyan color.
   - Text "kinder" below the snowflake in white (#ffffff), ~48px font-size, sans-serif
   - Tagline "kind, but with everything you actually need." below that in `hsl(185, 100%, 75%)` (accent-high), ~24px font-size

2. Use `sharp(Buffer.from(svgString)).png().toFile('public/og.png')` to render to PNG.

3. Run the script from kinder-site/ directory, verify public/og.png exists and is a valid PNG.

4. Delete the generator script after successful generation.

**Important:** Use absolute URL `https://kinder.patrykgolabek.dev/og.png` in all meta tags (not relative `/og.png`). Social crawlers require absolute URLs.

  <action>
Create `kinder-site/src/pages/404.astro` using the StarlightPage component wrapper so the 404 page inherits the site's dark theme and navigation.

The file must:

1. Import `StarlightPage` from `@astrojs/starlight/components/StarlightPage.astro` in the frontmatter fence.

2. Render `<StarlightPage frontmatter={{ title: 'Page Not Found', template: 'splash' }}>` as the root wrapper. Use `template: 'splash'` to avoid the sidebar on the 404 page.

3. Inside, create a centered content div with:
   - A large "404" heading styled with `color: var(--sl-color-accent)` and `font-size: 4rem`
   - A paragraph: "This page doesn't exist." at `font-size: 1.25rem`
   - A link to "/" with text "Back to Home" styled as a button: `display: inline-block; padding: 0.75rem 1.5rem; background: var(--sl-color-accent); color: var(--sl-color-black); border-radius: 0.5rem; text-decoration: none; font-weight: 600;`

4. Use inline styles (not a separate CSS file) — this is a single-use page. Reference Starlight CSS custom properties (`var(--sl-color-accent)`, `var(--sl-color-black)`) so it matches the dark theme.

Note: `disable404Route: true` was already set in Task 1, so there is no route collision.
