   - `Name() string` -- e.g., "container-runtime"
   - `Category() string` -- e.g., "Runtime"
   - `Platforms() []string` -- nil means all platforms, []string{"linux"} means Linux-only
   - `Run() []Result` -- returns one or more results (NOT error -- all error conditions expressed as warn/fail results)

   - `Name string json:"name"`

   - `Reason string json:"reason,omitempty"` -- WHY it matters (warn/fail only)

   - Iterates AllChecks()

   - Otherwise calls check.Run() and appends results
   - Returns ordered results (preserving insertion order from registry)

8. Helper `containsString(slice []string, s string) bool`
9. Helper `platformSkipMessage(platforms []string) string` -- returns "(linux only)" or "(linux/darwin only)"

Create `format.go` with:

1. Apache 2.0 copyright header
2. `FormatHumanReadable(w io.Writer, results []Result)` function per user's locked output style:
   - Group results by category preserving first-seen order (NOT alphabetical sort -- respect AllChecks() registry order)
   - For each category, print header: `\n=== CategoryName ===\n`
   - For each result in category, 2-space indent:
     - ok: `  ✓ Message\n` (Unicode checkmark U+2713)
     - fail: `  ✗ Name\n    Reason\n    → Fix\n` (Unicode x U+2717, arrow U+2192)
     - warn: `  ⚠ Name\n    Reason\n    → Fix\n` (Unicode warning U+26A0, arrow U+2192)
     - skip: `  ⊘ Name Message\n` (Unicode null U+2298, message is the platform tag like "(linux only)")
   - After all categories, print separator: `\n───\n` (three horizontal lines U+2500)
   - Print summary: `N checks: X ok, Y warning, Z failed, W skipped\n`
3. `FormatJSON(results []Result) map[string]interface{}` function returning envelope:
   - `"checks"`: the []Result slice directly (JSON tags handle serialization)
   - `"summary"`: map with total, ok, warn, fail, skip counts

5. Helper `countStatuses(results []Result) (ok, warn, fail, skip int)`
