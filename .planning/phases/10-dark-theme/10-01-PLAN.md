<task type="auto">
  <name>Task 1: Create theme CSS and register in Starlight config</name>
  <files>kinder-site/src/styles/theme.css, kinder-site/astro.config.mjs</files>
  <action>
1. Create the directory `kinder-site/src/styles/` if it does not exist.

2. Create `kinder-site/src/styles/theme.css` with the following content:

Dark mode overrides (`:root` selector — Starlight defaults to dark):
- `--sl-color-accent-low`: `hsl(185, 54%, 15%)` — deep cyan for badge backgrounds
- `--sl-color-accent`: `hsl(185, 100%, 45%)` — cyan for interactive elements (buttons, highlights)
- `--sl-color-accent-high`: `hsl(185, 100%, 75%)` — light cyan for link text (must pass 4.5:1 contrast against near-black bg)
- `--sl-color-black`: `hsl(220, 15%, 7%)` — deeper near-black background than Starlight default

Light mode overrides (`:root[data-theme='light']` selector):
- `--sl-color-accent-high`: `hsl(185, 80%, 22%)` — dark cyan for links on white background
- `--sl-color-accent`: `hsl(185, 90%, 40%)` — cyan for buttons/highlights
- `--sl-color-accent-low`: `hsl(185, 88%, 88%)` — light cyan tint for badge backgrounds

CRITICAL rules:
- Do NOT wrap any CSS in `@layer` — unlayered CSS automatically overrides Starlight's `@layer starlight.base` rules
- Do NOT use `!important` — unnecessary and creates specificity problems
- Do NOT override `--sl-color-bg` directly — it is an alias for `--sl-color-black`. Override `--sl-color-black` instead
- Do NOT override ThemeProvider or ThemeSelect components — they already handle FOUC prevention and toggle persistence

3. Update `kinder-site/astro.config.mjs` to add the `customCss` array to the `starlight()` config:
```javascript
customCss: [
  './src/styles/theme.css',
],
```
Place it after the `social` array in the starlight config object. Keep all existing config unchanged.

4. Run `cd kinder-site && npm run build` to verify the build succeeds.

5. Verify the custom CSS variables appear in the built output:
```bash
grep -r "sl-color-accent" kinder-site/dist/_astro/*.css | head -5
```
This confirms the CSS file was picked up and bundled.

6. Verify the background override is present:
```bash
grep -r "sl-color-black" kinder-site/dist/_astro/*.css | head -3
```
