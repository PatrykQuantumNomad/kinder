**Step 2: Replace the stub in envoygw.go with the complete Execute function.**

The new envoygw.go must:

1. Keep the existing copyright header and package declaration.
2. Add imports: `_ "embed"`, `"strings"`, `"sigs.k8s.io/kind/pkg/cluster/nodeutils"`, `"sigs.k8s.io/kind/pkg/errors"` (keep the existing `"sigs.k8s.io/kind/pkg/cluster/internal/create/actions"` import).
3. Add `//go:embed manifests/install.yaml` directive with `var envoyGWManifest string`.
4. Add a `const gatewayClassYAML` string containing the GatewayClass resource:
   ```
   apiVersion: gateway.networking.k8s.io/v1
   kind: GatewayClass
   metadata:
     name: eg
   spec:
     controllerName: gateway.envoyproxy.io/gatewayclass-controller
   ```
5. Keep existing `type action struct{}` and `NewAction()` unchanged.
6. Replace the Execute function body with this five-step sequence:
   - `ctx.Status.Start("Installing Envoy Gateway")` with `defer ctx.Status.End(false)`
   - Get control plane node: `ctx.Nodes()` -> `nodeutils.ControlPlaneNodes(allNodes)` -> `controlPlanes[0]` (same pattern as metallb.go and metricsserver.go)
   - **Step 1:** Apply install.yaml with `--server-side` flag: `node.Command("kubectl", "--kubeconfig=/etc/kubernetes/admin.conf", "apply", "--server-side", "-f", "-").SetStdin(strings.NewReader(envoyGWManifest)).Run()`. The `--server-side` flag is REQUIRED because the httproutes CRD is 372 KB which exceeds the 256 KB annotation limit for standard apply.
   - **Step 2:** Wait for certgen Job: `kubectl wait --namespace=envoy-gateway-system --for=condition=Complete job/eg-gateway-helm-certgen --timeout=120s`. This Job creates TLS Secrets required by the controller.
   - **Step 3:** Wait for controller Deployment: `kubectl wait --namespace=envoy-gateway-system --for=condition=Available deployment/envoy-gateway --timeout=120s`.
   - **Step 4:** Apply GatewayClass `eg` via `kubectl apply -f -` with `SetStdin(strings.NewReader(gatewayClassYAML))`. Do NOT use `--server-side` here (the GatewayClass is tiny).
   - **Step 5:** Wait for GatewayClass accepted: `kubectl wait --for=condition=Accepted gatewayclass/eg --timeout=60s`.
   - On success: `ctx.Status.End(true)` and `return nil`.
   - All errors wrapped with `errors.Wrap(err, "descriptive message")`.

Add a comment on the certgen Job name noting it was verified for v1.3.1 and should be re-checked on upgrade.

**Important:** Do NOT modify create.go. EGW-04 (disable flag) and EGW-05 (MetalLB warning) are already implemented there and require no changes.

Reference the complete Execute function in the research file (.planning/phases/05-envoy-gateway/05-RESEARCH.md, "Complete Execute Function" section) as a guide, but follow the established codebase patterns from metallb.go and metricsserver.go for formatting and style consistency.

**EGW-06 TLS termination documentation section:**

Include a dedicated section titled "## TLS Termination Guide (EGW-06)" that documents the manual cert path for TLS termination without cert-manager:

1. Generate a self-signed certificate:
   ```bash
   openssl req -x509 -sha256 -nodes -days 365 -newkey rsa:2048 \
     -keyout example.key -out example.crt \
     -subj "/CN=www.example.com/O=example"
   ```

2. Create the TLS Secret:
   ```bash
   kubectl create secret tls example-cert \
     --key=example.key \
     --cert=example.crt \
     --namespace=default
   ```

3. Reference in a Gateway HTTPS listener:
   ```yaml
   apiVersion: gateway.networking.k8s.io/v1
   kind: Gateway
   metadata:
     name: my-gateway
     namespace: default
   spec:
     gatewayClassName: eg
     listeners:
       - name: https
         protocol: HTTPS
         port: 443
         tls:
           mode: Terminate
           certificateRefs:
             - name: example-cert
               kind: Secret
   ```

4. Create an HTTPRoute targeting the HTTPS listener:
   ```yaml
   apiVersion: gateway.networking.k8s.io/v1
   kind: HTTPRoute
   metadata:
     name: my-https-route
     namespace: default
   spec:
     parentRefs:
       - name: my-gateway
         sectionName: https
     rules:
       - matches:
           - path:
               type: PathPrefix
               value: /
         backendRefs:
           - name: my-service
             port: 8080
   ```
