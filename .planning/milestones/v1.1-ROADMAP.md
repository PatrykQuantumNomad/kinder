## Milestone Summary

**Key Decisions:**
- Astro + Starlight for website (Rationale: Static site generator, fast, great for docs + landing pages)
- Dark-only mode, no light theme (Rationale: Terminal aesthetic is core identity)
- npm over pnpm (Rationale: GitHub Actions CI compatibility)
- `make install` as only install method (Rationale: Binary distribution unconfirmed)
- Starlight CSS custom properties over Tailwind (Rationale: Sufficient, Tailwind v4 unstable)
- Separate kinder-site/ directory (Rationale: Clean separation from Go codebase)

**Issues Resolved:**
- GitHub org updated from patrykattc to PatrykQuantumNomad
- Kubernetes Dashboard references corrected to Headlamp throughout docs
- Dark mode FOUC prevented with inline script + dark-only enforcement

**Issues Deferred:**
- Animated terminal demo (ENH-01)
- Blog section for release announcements (ENH-02)
- Contributing guide page (ENH-03)
- Changelog page auto-generated from git history (ENH-04)

**Technical Debt Incurred:**
- Orphaned houston.webp (96KB Astro scaffold default) — never referenced, safe to delete
- Minor: base64 -d vs base64 --decode inconsistency across docs (cosmetic only)
