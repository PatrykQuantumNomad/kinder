# Milestone v2.1: Known Issues & Proactive Diagnostics

**Status:** SHIPPED 2026-03-06
**Phases:** 38-41
**Total Plans:** 10

## Overview

Address Kind's documented known issues by expanding `kinder doctor` with 13 new diagnostic checks (bringing total to 18), adding automatic mitigations during cluster creation where safe, and documenting all checks on the website.

## Phases

### Phase 38: Check Infrastructure and Interface

**Goal**: Developers can run `kinder doctor` and see all existing checks executing through a unified Check interface with category-grouped output and platform filtering
**Depends on**: Phase 37 (v2.0 complete)
**Plans**: 2 plans

Plans:
- [x] 38-01: Check interface, Result type, registry, platform filtering, output formatters, SafeMitigation skeleton
- [x] 38-02: Migrate 3 existing checks to Check interface and refactor doctor.go CLI layer

**Details:**
- Check interface with Name(), Category(), Platforms(), Run() methods
- Result type with 6 JSON-tagged fields (ok/warn/fail/skip statuses)
- AllChecks() explicit registry with RunAllChecks() centralized platform filtering
- FormatHumanReadable() category-grouped Unicode output and FormatJSON() envelope formatter
- SafeMitigation struct with NeedsFix/Apply/NeedsRoot fields and ApplySafeMitigations() skeleton
- All 3 existing checks (container runtime, kubectl, NVIDIA GPU) migrated as 5 individual checks
- doctor.go CLI reduced from 298 to 74 lines

### Phase 39: Docker and Tool Configuration Checks

**Goal**: Users running `kinder doctor` get actionable warnings about the five most common Docker/tool configuration problems
**Depends on**: Phase 38 (Check interface and registry must exist)
**Plans**: 2 plans

Plans:
- [x] 39-01: Disk space, daemon.json, and Docker snap checks (DOCK-01, DOCK-02, DOCK-03)
- [x] 39-02: Kubectl version skew and Docker socket permission checks (DOCK-04, DOCK-05)

**Details:**
- diskSpaceCheck warns at <5GB, fails at <2GB using build-tagged statfs (linux/darwin)
- daemonJSONCheck detects "init: true" across 6+ candidate paths
- dockerSnapCheck resolves symlinks to detect /snap/ installations
- kubectlVersionSkewCheck parses `kubectl version --client -o json` with +/-1 minor tolerance
- dockerSocketCheck detects permission denied with usermod fix suggestion
- allChecks registry expanded from 5 to 10 checks

### Phase 40: Kernel, Security, and Platform-Specific Checks

**Goal**: Linux users running `kinder doctor` get advance warnings about kernel limits, security modules, and platform configurations
**Depends on**: Phase 39 (all checks share testing patterns)
**Plans**: 3 plans

Plans:
- [x] 40-01: Inotify limits and kernel version checks (KERN-01, KERN-04)
- [x] 40-02: AppArmor, SELinux, and firewalld checks (KERN-02, KERN-03, PLAT-01)
- [x] 40-03: WSL2 detection, rootfs device check, and allChecks registry wiring (PLAT-02, PLAT-04)

**Details:**
- inotifyCheck warns on low max_user_watches (<524288) and max_user_instances (<512) with sysctl fix commands
- kernelVersionCheck fails on < 4.6 (cgroup namespace hard blocker)
- AppArmor and SELinux checks are independent (LSM stacking since kernel 5.1)
- SELinux warns only on Fedora; firewalld defaults to nftables when config absent
- WSL2 multi-signal detection (microsoft in /proc/version + corroborating evidence) prevents Azure VM false positives
- Cgroup v2 controller verification (cpu, memory, pids) when WSL2 confirmed
- Rootfs device check for BTRFS storage driver detection
- allChecks expanded from 10 to 17 checks

### Phase 41: Network, Create-Flow Integration, and Website

**Goal**: Users get subnet clash detection, automatic safe mitigations during cluster creation, and comprehensive Known Issues documentation
**Depends on**: Phase 40 (all checks must exist before create-flow integration)
**Plans**: 3 plans

Plans:
- [x] 41-01: Subnet clash detection check with TDD (PLAT-03)
- [x] 41-02: ApplySafeMitigations wiring into create flow (INFRA-05)
- [x] 41-03: Known Issues documentation page on website (SITE-01)

**Details:**
- subnetClashCheck detects Docker/host subnet overlaps using netip.Prefix.Overlaps
- normalizeAbbreviatedCIDR handles macOS netstat -rn abbreviated destination formats
- Self-referential Docker routes skipped by exact netip.Prefix equality
- ApplySafeMitigations called between containerd config patches and p.Provision()
- Mitigation errors logged as warnings, never fatal
- 487-line Known Issues page documenting all 18 checks across 8 categories
- What/Why/Platforms/Fix consistent four-section structure
- Sidebar entry and cross-links with Troubleshooting page

## Milestone Summary

**Key Decisions:**
- allChecks initialized as []Check{} (not nil) for non-nil guarantee
- FormatJSON returns map[string]interface{} for flexible JSON serialization
- ApplySafeMitigations early-returns on non-Linux platforms
- Inline fakeCmd test doubles instead of importing testutil from pkg/cluster/internal/
- Build-tagged statfsFreeBytes with int64 cast for macOS/Linux Bsize portability
- daemonJSONCheck searches 6 candidate paths including Windows ProgramData
- referenceK8sMinor = 31 as constant, +/-1 minor skew tolerance
- AppArmor and SELinux checks completely independent (LSM stacking since kernel 5.1)
- SELinux warns only on Fedora; firewalld defaults to nftables when absent
- WSL2 detection requires two signals to prevent Azure VM false positives
- Subnet clash: IPv4 only, self-referential Docker routes skipped
- Mitigation errors are warnings, never block cluster creation
