# Milestone v2.2: Cluster Capabilities

**Status:** SHIPPED 2026-04-10
**Phases:** 42-46
**Total Plans:** 14

## Overview

Add four cluster capabilities to kinder — multi-version per-node Kubernetes validation, offline/air-gapped cluster creation, local-path-provisioner dynamic storage, host-directory mounting pre-flight — plus a provider-abstracted `kinder load images` subcommand that ties the offline and multi-version workflows together. All features integrate using packages already in `go.mod` with zero new dependencies. Doctor registry expanded from 18 to 23 checks.

## Phases

### Phase 42: Multi-Version Node Validation

**Goal**: Users can configure per-node Kubernetes versions and kinder validates version-skew correctness at config parse time instead of surfacing cryptic kubeadm errors after provisioning begins
**Depends on**: Phase 41
**Requirements**: MVER-01, MVER-02, MVER-03, MVER-04, MVER-05
**Plans**: 2 plans

Plans:
- [x] 42-01: ExplicitImage sentinel, fixupOptions fix, version-skew config validation
- [x] 42-02: Doctor cluster-skew check, get nodes VERSION/IMAGE/SKEW columns

**Details:**

- Added `ExplicitImage bool` to internal `Node` struct, captured **pre-defaults** in `encoding/convert.go` because `SetDefaultsCluster` fills empty Image fields before conversion runs
- `fixupOptions` in `create.go` now skips nodes with `ExplicitImage=true` when applying `--image` flag override — preserves per-node images in config
- `validateVersionSkew(nodes []Node)` in `validate.go` parses semver tags via `version.ParseSemantic`; non-semver tags (e.g. `"latest"`) skip validation entirely for test/dev backward compat
- HA control-plane nodes required to share minor version (etcd consistency); workers required within 3 minor versions of CP; violations collected and reported as a table with remediation hints
- `clusterNodeSkewCheck` doctor check (`pkg/internal/doctor/clusterskew.go`) with injectable `listNodes`/`kubeVersion` deps for test isolation; detects HA CP mismatch, workers >3 behind, config drift (image tag vs live `/kind/version`), and version-read failures
- `ComputeSkew(cpMinor, nodeMinor uint) (string, bool)` pure exported function on `kinder get nodes`: exact match → `✓`, any difference → `✗ (±N)`, ok=true only when ≤3 behind
- `kinder get nodes` extended to tabwriter table with NAME/ROLE/STATUS/VERSION/IMAGE/SKEW columns; JSON encoder includes all fields
- AllChecks registry count moved from 18 to 19 (added `cluster-node-skew`)

- [x] 44-03: Doctor CVE-2025-62878 check: warns when local-path-provisioner < v0.0.34

**Details:**

- `RequiredAddonImages` `LocalPath` branch adds both `docker.io/rancher/local-path-provisioner:v0.0.35` and `docker.io/library/busybox:1.37.0` to the required image set

### Phase 45: Host-Directory Mounting

**Goal**: Users can mount host directories into cluster nodes with clear pre-flight validation, explicit platform warnings, and documented guidance for wiring mounts through to pods via hostPath PVs
**Depends on**: Phase 44
**Requirements**: MOUNT-01, MOUNT-02, MOUNT-03, MOUNT-04
**Plans**: 4 plans (including SC-3 gap closure)

Plans:
- [x] 45-01: Host path pre-flight validation and platform propagation warnings
- [x] 45-02: Doctor host-mount-path and docker-desktop-file-sharing checks
- [x] 45-03: Host directory mounting documentation guide
- [x] 45-04: Gap closure — wire doctor mount checks to cluster config via `--config` flag

**Details:**

- `validateExtraMounts(cfg *config.Cluster) error` in `create.go` walks every node's ExtraMounts, resolves relative paths via `filepath.Abs`, calls `os.Stat` to verify host path exists — runs **between `opts.Config.Validate()` and `p.Provision()`** so any failure aborts cleanly before any container is created
- Structured error identifies which node index and mount index failed
- `logMountPropagationPlatformWarning(logger, cfg)` on darwin/windows emits single warning when any extraMount specifies non-None propagation mode (`HostToContainer`/`Bidirectional`) — uses warn-once return-after-first-match pattern consistent with `logMetalLBPlatformWarning`
- `hostMountPathCheck` and `dockerDesktopFileSharingCheck` added to `pkg/internal/doctor/hostmount.go` with injected `getMountPaths`/`statPath` deps for test isolation
- `dockerDesktopFileSharingCheck` reads `settings-store.json` and falls back to Docker Desktop default dirs (`/Users`, `/Volumes`, `/private`, `/tmp`, `/var/folders`) when the settings file is absent or malformed
- `isPathCovered` uses `dir+"/"` separator + exact equality to prevent `/Users` matching `/Userspace/data`
- Both mount checks registered under new "Mounts" category; AllChecks registry count moved 21→23
- Two-hop host-to-pod guide published at `kinder-site/src/content/docs/guides/host-directory-mounting.md` with complete runnable example (cluster config + PV + PVC + Pod + `kubectl logs` verification), text-based flow diagram, and platform notes for macOS/Windows/Linux with troubleshooting blocks for PVC stuck in Pending, empty /data, and path-not-found
- **Plan 45-04 gap closure (SC-3 fix):** `mountPathConfigurable` unexported interface in `doctor/check.go` plus exported `SetMountPaths([]string)` that type-asserts each check in `allChecks`; both mount checks gained `setMountPaths` methods; `kinder doctor` gained `--config` flag, `extractMountPaths` helper (deduplicates by seen map for multi-node configs with shared host paths), and `runE` wiring that calls `encoding.Load(flags.Config)` then `doctor.SetMountPaths(paths)` before `RunAllChecks`; `SetMountPaths(nil)` restores skip behavior for backward compat

### Phase 46: kinder load images Command

**Goal**: Users can load one or more local images into all nodes of a running cluster with a single command that works across all three providers and handles Docker Desktop 27+ containerd image store compatibility
**Depends on**: Phase 43
**Requirements**: LOAD-01, LOAD-02, LOAD-03, LOAD-04
**Plans**: 2 plans

Plans:
- [x] 46-01: LoadImageArchiveWithFallback in nodeutils for Docker Desktop 27+ containerd fallback
- [x] 46-02: images subcommand with provider-abstracted save/inspect, smart-load, load.go registration

**Details:**

- `LoadImageArchiveWithFallback(nodes, openArchive func() (io.ReadCloser, error))` in `pkg/cluster/nodeutils/util.go` — factory-pattern reader because tar streams are consumed on read and cannot be rewound; two-attempt fallback: attempt 1 uses `ctr images import --all-platforms`, attempt 2 retries without `--all-platforms` only if attempt 1 fails with `isContentDigestError` (Docker Desktop 27+ containerd image store `content digest: not found` failure); all other errors return immediately
- `isContentDigestError` checks `exec.RunError.Output` (combined stdout+stderr) first, falls back to `err.Error()` string scan — uses `stderrors` alias (stdlib `"errors"`) to avoid conflict with existing `sigs.k8s.io/kind/pkg/errors` import
- `runImport` unexported helper encapsulates `ctr images import` invocation; `LoadImageArchive` left unchanged to preserve existing `image-archive` command API
- 5-case `TestIsContentDigestError` table-driven test covering RunError.Output path, string fallback, happy path, and edge cases
- `pkg/cmd/kind/load/images/images.go` — new `kinder load images` subcommand implementing:
  - **Provider abstraction:** `providerBinaryName()` reads `KIND_EXPERIMENTAL_PROVIDER` directly for finch/nerdctl.lima variants because `provider.Name()` always returns "nerdctl"; for docker/podman, `provider.Name()` IS the binary
  - **Smart-load skip (LOAD-04):** before saving any tar, check each candidate node — if image ID and tag already match, skip the node with log message; if image ID present but tag missing, re-tag in place; only nodes truly lacking the image are loaded
  - **Docker Desktop 27+ fallback (LOAD-02/03):** `loadImage()` calls `nodeutils.LoadImageArchiveWithFallback` with `os.Open` factory closure — factory invoked twice for the two-attempt fallback
  - `save()` and `imageID()` take `binaryName` parameter — no hardcoded `docker` at any call site
  - `removeDuplicates`, `checkIfImageReTagRequired`, `sanitizeImage` copied from docker-image package (package-private there, no import cycle alternative)
- `pkg/cmd/kind/load/load.go` registers `images.NewCommand` as the third `load` subcommand alongside `docker-image` and `image-archive`

## Milestone Summary

**Key Decisions:**

- ExplicitImage sentinel captured pre-defaults (post-defaults detection impossible once SetDefaultsCluster fills Image)
- Skip version-skew check on non-semver tags (preserves backward compat with "latest")
- AirGapped propagated via ClusterOptions → Config, matching CreateWithRetain pattern
- Accumulate-all-missing images (not fail-on-first) so users get one actionable error
- `RequiredAddonImages` imports addon packages in common (addons don't import common → zero cycle risk)
- Inline `allAddonImages` in offlinereadiness.go (importing pkg/cluster/internal creates doctor import cycle)
- LocalPath uses opt-out boolVal (default true), consistent with MetalLB/CertManager batteries-included pattern
- StorageClass renamed `local-path` to avoid collision with legacy `installstorage` `standard` class
- busybox pinned to 1.37.0 with IfNotPresent for air-gap PVC compatibility
- CVE-2025-62878 threshold v0.0.34 returns ok (it's the fix version); only strictly less-than warns
- `validateExtraMounts` runs between Validate and Provision for clean rollback-free failure
- `mountPathConfigurable` interface is unexported; doctor package owns wiring, cmd layer only sees exported SetMountPaths
- `providerBinaryName` reads KIND_EXPERIMENTAL_PROVIDER because provider.Name() always returns "nerdctl"
- `LoadImageArchiveWithFallback` uses factory-pattern reader (tar streams can't be rewound)
- `stderrors` alias for stdlib errors.As avoids conflict with sigs.k8s.io/kind/pkg/errors import
- Container-exec kubectl for CVE version probe avoids pkg/cluster/internal import cycle (same as realListNodes)
- **Zero new Go module dependencies across all 5 features**

**Issues Resolved:**

- SC-3 gap in Phase 45: doctor mount checks were stubbed to always skip because `getMountPaths` defaulted to `nil`. Plan 45-04 closed the gap by adding `--config` flag + `mountPathConfigurable` interface + `SetMountPaths` wiring
- Documentation location drift (Phase 43): working-offline.md was initially written to legacy `site/content/docs/user/` (upstream kind site, not deployed). Resolved by porting to `kinder-site/src/content/docs/guides/working-offline.md` in docs polish commit `1b4c29d7` and deleting stale file
- REQUIREMENTS.md checkbox drift: LOAD-03 was unchecked despite Phase 46 completion (fixed in `1b4c29d7`); STOR-01..06 checkboxes and Traceability table marked `[ ]` / "Pending" despite Phase 44 completion (fixed during audit)
- AllChecks registry count test races: Phases 44-02 and 44-03 ran out of plan order; parallel executor resolved duplicate count-assertion updates in a single auto-fix commit

**Issues Deferred:**

- Phase 43 human verification (4 items): live runtime testing of air-gapped success/fail across docker/podman/nerdctl, addon warning output, doctor offline-readiness live run. All code paths unit-tested via dependency injection; binary rebuilt and tested in ship
- Phase 44 human verification (2 items): live cluster testing of single-node PVC auto-binding and multi-node WaitForFirstConsumer binding. Dynamic provisioner binding inherently requires runtime testing; code wiring complete
- Phase 46 human verification: Docker provider approved after fresh binary test; Podman/Finch/Docker Desktop 27+ fallback/smart-load re-run remain untested on user's environment but not ship-blocking
- **MOUNT-05/06** (auto-PV from extraMounts), **AIRGAP-07/08** (offline presets, `kinder build bake-images`), **MVER-06** (`--profile upgrade-test`) deferred to v2.3+
