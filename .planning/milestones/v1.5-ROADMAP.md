- Core vs optional addon grouping (MetalLB/Metrics Server/CoreDNS = core; Envoy Gateway/Headlamp/Local Registry/cert-manager = optional)

- Tutorial structure: overview, prerequisites, numbered steps, cleanup with Expected output blocks
- kinder env documented as single JSON object (not array)
- ci profile = MetricsServer + CertManager only (no MetalLB)
