# Requirements: Kinder v2.2 Cluster Capabilities

**Defined:** 2026-04-08
**Core Value:** A single command gives developers a local Kubernetes cluster where LoadBalancer services, Gateway API routing, metrics, and dashboards all work without any manual setup.

## v2.2 Requirements

Requirements for v2.2 Cluster Capabilities. Each maps to roadmap phases.

### Multi-Version Clusters

- [x] **MVER-01**: `--image` flag only overrides nodes without an explicit image in config (fix fixupOptions bug) — Phase 42-01, ExplicitImage sentinel captured pre-defaults
- [x] **MVER-02**: Config validation rejects invalid version-skew combinations before provisioning (control-plane same version, workers within 3-minor of control-plane) — Phase 42-01, validateVersionSkew
- [x] **MVER-03**: Clear error messages surface version-skew violations at config parse time instead of cryptic kubeadm join failures — Phase 42-01, table-format error output
- [x] **MVER-04**: `kinder doctor` detects version-skew issues in running clusters — Phase 42-02, clusterNodeSkewCheck
- [x] **MVER-05**: `kinder get nodes` output includes per-node Kubernetes version column — Phase 42-02, tabwriter with VERSION/IMAGE/SKEW columns

### Air-Gapped Clusters

- [x] **AIRGAP-01**: `--air-gapped` flag on `kinder create cluster` enables offline mode — Phase 43-01
- [x] **AIRGAP-02**: Air-gapped mode fails fast with clear error listing missing images instead of retrying pulls — Phase 43-02, accumulate-all-missing fast-fail
- [x] **AIRGAP-03**: All three providers (docker/podman/nerdctl) support air-gapped mode via ProvisionOptions — Phase 43-02
- [x] **AIRGAP-04**: Addon image warning output lists all required images that need pre-loading before creation — Phase 43-01, 43-02
- [x] **AIRGAP-05**: `kinder doctor` offline readiness check lists which required images are missing — Phase 43-03, offlineReadinessCheck
- [x] **AIRGAP-06**: Two-mode design documented (pre-create bake via privileged container commit vs. post-create load) — Phase 43-03, working-offline.md (ported to kinder-site/ during docs polish)

### Local Storage (local-path-provisioner)

- [x] **STOR-01**: local-path-provisioner v0.0.35 installed as a default addon via `installlocalpath` action — Phase 44-01
- [x] **STOR-02**: `local-path` is the default StorageClass; `installstorage` gated out when LocalPath enabled — Phase 44-01
- [x] **STOR-03**: Embedded manifest patches busybox to `busybox:1.37.0` with `imagePullPolicy: IfNotPresent` — Phase 44-01
- [x] **STOR-04**: `Addons.LocalPath *bool` field added to v1alpha4 config (full 5-location pipeline) — Phase 44-01
- [x] **STOR-05**: `kinder doctor` CVE-2025-62878 check warns if running cluster has vulnerable local-path-provisioner — Phase 44-03
- [x] **STOR-06**: PVCs dynamically provision and bind automatically in single and multi-node clusters — Phase 44-01 code wiring complete; runtime binding deferred to live-cluster human verification (not ship-blocking)

### Host Directory Mounting

- [x] **MOUNT-01**: Pre-flight validation checks host path existence before node provisioning with clear error message — Phase 45-01, validateExtraMounts between Validate() and Provision()
- [x] **MOUNT-02**: Platform warning emitted when non-None propagation mode specified on macOS/Windows — Phase 45-01, logMountPropagationPlatformWarning
- [x] **MOUNT-03**: `kinder doctor` checks host path exists and Docker Desktop file sharing is configured (macOS) — Phase 45-02 + Phase 45-04 SC-3 gap closure (--config flag wiring via mountPathConfigurable interface)
- [x] **MOUNT-04**: Documentation covers two-hop mount pattern (host → node extraMount → pod hostPath PV) — Phase 45-03, kinder-site/guides/host-directory-mounting.md

### Load Images Command

- [x] **LOAD-01**: `kinder load images` subcommand loads one or more images into all cluster nodes — Phase 46-02
- [x] **LOAD-02**: Provider-abstracted image saving (works with docker/podman/nerdctl, not hardcoded to docker save) — Phase 46-02, providerBinaryName + save()/imageID() binaryName parameter
- [x] **LOAD-03**: Fallback import strategy for Docker Desktop 27+ containerd image store compatibility — Phase 46-01, LoadImageArchiveWithFallback with factory-pattern reader
- [x] **LOAD-04**: Smart-load skips images already present on nodes — Phase 46-02, per-node image ID + tag check with re-tag in place

## Out of Scope

| Feature | Reason |
|---------|--------|
| Bundling addon images into kinder binary | Binary would become multi-GB; breaks Homebrew distribution |
| Auto-detecting internet availability | Unreliable on VPNs/proxies; causes confusing silent behavior |
| Auto-rolling upgrades via multi-version clusters | Out of scope; kubeadm covers this already |
| HA control-plane version skew > 1 minor | kubeadm refuses it; kinder must validate and reject |
| OCI Go libraries for image operations | All image operations go through Docker/containerd CLI (established pattern) |
| Auto-PV from extraMounts (v2.2) | Type system complexity for limited gain; defer to v2.3 |
