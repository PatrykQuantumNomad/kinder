# Milestone v1.0: Batteries Included

**Status:** SHIPPED 2026-03-01
**Phases:** 1-8
**Total Plans:** 12

## Overview

Starting from the upstream kind fork, this roadmap built kinder's batteries-included cluster experience in eight phases. Phase 1 laid the shared infrastructure (config schema and action scaffolding) that every subsequent phase depended on. Phases 2-6 each delivered one complete addon — MetalLB, Metrics Server, CoreDNS tuning, Envoy Gateway, and Dashboard — in dependency order. Phase 7 validated the full system with cross-addon integration tests. Phase 8 closed audit gaps from milestone verification. All eight phases complete, `kinder create cluster` produces a fully functional development cluster with no manual setup required.

## Milestone Summary

**Key Decisions:**
- Fork kind, don't wrap it — full control over action pipeline, single binary
- *bool for v1alpha4 Addons fields — nil means not-set, defaults to true; avoids Go bool zero-value ambiguity
- Embed all addon manifests at build time via go:embed — offline-capable, no external tools
- MetalLB must be fully ready before Envoy Gateway actions run — hard ordering dependency
- CoreDNS patching uses read-modify-write in Go, not kubectl patch — Corefile is a string blob
- Headlamp v0.40.1 instead of kubernetes/dashboard — archived Jan 2026, Helm dependency
- --server-side apply for Envoy Gateway CRDs — httproutes CRD exceeds 256 KB annotation limit

**Issues Resolved:**
- All-false addon guard silently overriding explicit user config (Phase 8)
- Integration script missing kubectl context selection (Phase 8)

**Issues Deferred:**
- Podman rootless MetalLB viability (L2 speaker + raw sockets) needs real-world testing

**Technical Debt Incurred:**
- Runtime behaviors (pod readiness, LoadBalancer IP assignment, DNS, dashboard UI) only verified via human testing, not automated CI
