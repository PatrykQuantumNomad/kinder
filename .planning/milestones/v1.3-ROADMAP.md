Fix critical bugs, reduce provider code duplication, and add local registry, cert-manager addons and CLI diagnostic tools.

## Phases

**Depends on**: Nothing (first v1.3 phase)
**Plans**: 2 plans

Plans:
- [x] 19-01: Fix port leak (BUG-01) and tar truncation (BUG-02)

## Milestone Summary

**Key Decisions:**
- Extract shared provider code to common/ package: eliminate ~70-80% duplication, prevent drift bugs
- Local registry as addon (not shell script): consistent with batteries-included ethos
- Cert-manager alongside Envoy Gateway: natural pairing; TLS commonly needed with Gateway API
- registry:2 not registry:3: kind ecosystem on v2, v3 deprecated storage drivers
- ContainerdConfigPatches before Provision: cannot inject post-provisioning
- cert-manager v1.16.3 (v1.17.6 not yet released at time of implementation)
- Provider.Name() via fmt.Stringer: type assertion instead of new interface method

**Issues Resolved:**
- Port leak under early exit in generatePortMappings (BUG-01)
- Silent data loss on truncated tar files (BUG-02)
- Inconsistent cluster name resolution in ListInternalNodes (BUG-03)
- Non-deterministic network sort from weak ordering violation (BUG-04)
- ~70-80% provider code duplication between docker/nerdctl/podman

**Issues Deferred:**
- Pull-through cache for local registry (REG-05)
- Registry web UI (REG-06)
- trust-manager addon (CERT-05)
- ACME/Let's Encrypt issuer support (CERT-06)
- kinder env --shell fish (CLI-05)
- kinder doctor --fix auto-remediation (CLI-08)

**Technical Debt Incurred:**
- Pre-existing TODO: "picking ports locally is less than ideal..." in docker/create.go:61 and nerdctl/create.go:61
- Pre-existing TODOs at provider.go:90 ("consider breaking API") and :249 ("ListNodes refactor")
- Internal SetDefaultsCluster does not set Addons.LocalRegistry/CertManager defaults (v1alpha4 pipeline covers all user flows)
