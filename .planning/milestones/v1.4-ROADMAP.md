# Milestone v1.4: Code Quality & Features

**Status:** ✅ SHIPPED 2026-03-04
**Phases:** 25-29
**Total Plans:** 13

## Overview

Clean up remaining code quality issues, improve architecture, add unit tests for addon actions, and deliver new developer-facing features. Every change is independently buildable and verifiable with `go build ./... && go test ./...`.

### Phase 26: Architecture

**Goal**: context.Context flows from create.go through ActionContext into every addon Execute() call and waitforready loop; a centralized AddonEntry registry replaces hard-coded runAddon calls in create.go
**Depends on**: Phase 25
**Plans**: 2 plans

Plans:
- [x] 26-01: Context field in ActionContext + AddonEntry registry loop
- [x] 26-02: CommandContext propagation in all addons + context-aware waitforready

**Details:**
- context.Context added to ActionContext struct (not function parameter — minimizes call-site churn)
- 7 hard-coded runAddon() calls replaced with data-driven []AddonEntry registry loop
- All 7 addon Execute() methods use node.CommandContext(ctx.Context, ...) instead of node.Command(...)
- tryUntil uses select on ctx.Done() with 500ms fallback for immediate cancellation
- context.Background() at create.go call site — signal-wired context deferred to future phase

### Phase 28: Parallel Execution

**Depends on**: Phase 26, Phase 27
**Plans**: 2 plans

Plans:
- [x] 28-01: sync.OnceValues Nodes() cache fix + golang.org/x/sync dependency + race test
- [x] 28-02: Wave-based parallel addon execution with errgroup, DAG docs, timing summary, Makefile test-race

**Details:**
- RWMutex-based cachedData replaced with sync.OnceValues for exactly-once node caching

- Wave 2: EnvoyGateway sequential after g.Wait() (depends on MetalLB)

**Depends on**: Phase 25 (clean baseline); Phase 28 (per-addon result data available)
**Plans**: 2 plans

Plans:
- [x] 29-01: --output json on env, doctor, and get clusters commands
- [x] 29-02: --output json on get nodes + --profile flag on create cluster

**Details:**
- Consistent flagpole/switch/json.NewEncoder pattern across all 4 commands

- 4 presets: minimal (no addons), full (all addons), gateway (MetalLB + EnvoyGateway), ci (MetalLB + Metrics Server)

## Milestone Summary

**Key Decisions:**
- Context in struct (not function param) — minimal call-site churn trade-off
- Wave-based parallel not full DAG — 7 addons with shallow deps; DAG adds 200+ lines for zero benefit
- go directive settled at 1.24.0 (toolchain go1.26 enforces minimum)
- sync.OnceValues over RWMutex — eliminates TOCTOU race, single-call guarantee
- errgroup.SetLimit(3) — bounds concurrent kubectl apply calls
- Consistent flagpole/switch/json.NewEncoder pattern for all JSON commands

**Issues Resolved:**
- Layer violation: cluster package imported from CLI layer (fixed by kindversion internal pkg)
- 55+ golangci-lint violations across 60+ files (fixed during v2 migration)
- TOCTOU race in Nodes() cache (fixed with sync.OnceValues)
- SHA-1 usage for subnet generation (replaced with SHA-256)

**Issues Deferred:**
- context.Background() at create.go call site — signal-wired context for future phase
- Misleading comment in create.go about nodesOnce cache sharing (cosmetic)

**Technical Debt Incurred:**
- doctor.go nil slice vs empty slice for zero-result edge case (non-blocking; always ≥2 checks)
- 9 runtime integration tests need human verification (require Docker + node image)
